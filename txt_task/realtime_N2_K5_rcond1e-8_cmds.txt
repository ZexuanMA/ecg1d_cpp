Task: Real-time TDVP, N=2 K=5, full HO period (T = 2*pi), dt = 1e-3, rcond = 1e-8

Purpose
- The original N=2 K=5 run (default rcond=1e-4) had cond(C)~3.8e9 from
  step 1, sv_min stuck at the Tikhonov floor 1e-8, and broke down within
  50 RK4 steps: norm crashed to 0.06, raw E*norm drifted 14x, parity
  badly broken.
- Hypothesis: bosonic permutation symmetry creates rank-deficient C from
  the start, and rcond=1e-4 truncates ~5 decades below the genuinely
  zero modes. Lowering rcond to 1e-8 should keep all the physically
  active singular directions.
- If this fixes the dynamics, the N=2 K=5 failure was an SVD-tuning
  problem, not a fundamental basis-size problem.
- If it does NOT fix it, the manifold is irreducibly rank-deficient and
  we'd need either lambda_C ~ 1e-12 or a different (non-redundant)
  ansatz.
- NOT enabled: --enforce-norm, --u-split-trotter (per user request).

Note on output filenames: the original N=2 K=5 run was killed before
the trace CSV writer ran, so there is nothing to back up. This run will
write the first complete N=2 K=5 trace.

Pre-flight: pick a free core (filtered to your allowed set)

ALLOWED=$(taskset -pc $$ | awk -F'list: ' '{print $2}')
echo "your allowed cores: $ALLOWED"

top -bn2 -d 1 -1 2>/dev/null | awk -v allowed="$ALLOWED" '
  BEGIN {
    n=split(allowed, parts, ",")
    for (i=1;i<=n;i++) {
      if (split(parts[i], r, "-") == 2) for (c=r[1]+0;c<=r[2]+0;c++) ok[c]=1
      else ok[parts[i]+0]=1
    }
    snap=0
  }
  /^top - / { snap++ }
  snap==2 && /^%Cpu[0-9]/ {
    c=$1; gsub(/^%Cpu/,"",c); gsub(/[^0-9]/,"",c)
    if ((c+0) in ok) {
      for (i=1;i<=NF;i++) if ($i=="id,") { idle=$(i-1); break }
      printf "cpu %3d  used=%5.1f%%  idle=%5.1f%%\n", c, 100-idle, idle
    }
  }
' | sort -k3 -t= -nr
