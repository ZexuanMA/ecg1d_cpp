Task: Real-time TDVP, N=1 K=5, full HO period (T = 2*pi), dt = 1e-3, Wiener filter OFF (--no-wiener)
       — runs CONCURRENTLY with the rcond=1e-6 job, writing to a separate sandbox cwd.

Purpose
- Repeat N=1 K=5 Step-4 with --no-wiener and compare against the
  Wiener-on baselines (default-rcond and rcond=1e-6).
- The user wants results NOW, in parallel with the still-running
  rcond=1e-6 job (PID 3618072, core 118), without touching the main
  out/verify/ outputs that the rcond=1e-6 run is generating.

How separation works (READ THIS)
- The binary writes its CSVs via out_path(), which is hard-coded to
  the relative directory "out/verify".  There is NO CLI override.
- Therefore output paths are determined entirely by the cwd of the
  process.  By running the --no-wiener job from a sandbox cwd
  (runs/no_wiener_N1K5/), its CSVs land at
      runs/no_wiener_N1K5/out/verify/step4_ecg_*_N1_K5.csv
  while the rcond=1e-6 job, started from the project root, keeps
  writing to
      out/verify/step4_ecg_*_N1_K5.csv
  The two output sets never collide.
- Step 4 needs the cached Step-1 basis as input
  (out/verify/step1_ecg_basis_N1_K5.csv), so the sandbox symlinks
  that file in.
- The binary path is given as an ABSOLUTE path so the sandboxed cd
  does not break it.
- Logs still live in the project's logs/ directory (absolute path in
  the redirect) so you do not have to hunt for them.

Pre-flight: pick a free core (filtered to your allowed set)

# 1. Your allowed cores (cgroup / affinity)
ALLOWED=$(taskset -pc $$ | awk -F'list: ' '{print $2}')
echo "your allowed cores: $ALLOWED"

# 2. Per-allowed-core usage; cores not listed are 100% idle.
top -bn2 -d 1 -1 2>/dev/null | awk -v allowed="$ALLOWED" '
  BEGIN {
    n=split(allowed, parts, ",")
    for (i=1;i<=n;i++) {
      if (split(parts[i], r, "-") == 2) for (c=r[1]+0;c<=r[2]+0;c++) ok[c]=1
      else ok[parts[i]+0]=1
    }
    snap=0
  }
  /^top - / { snap++ }
  snap==2 && /^%Cpu[0-9]/ {
    c=$1; gsub(/^%Cpu/,"",c); gsub(/[^0-9]/,"",c)
    if ((c+0) in ok) {
      for (i=1;i<=NF;i++) if ($i=="id,") { idle=$(i-1); break }
      printf "cpu %3d  used=%5.1f%%  idle=%5.1f%%\n", c, 100-idle, idle
    }
  }
' | sort -k3 -t= -nr

How to run

cd /home/gyqyan/zexuan/ecg1d_cpp

# 1) Build the sandbox cwd, symlink the Step-1 basis cache in.
mkdir -p runs/no_wiener_N1K5/out/verify logs
ln -sf /home/gyqyan/zexuan/ecg1d_cpp/out/verify/step1_ecg_basis_N1_K5.csv \
       runs/no_wiener_N1K5/out/verify/step1_ecg_basis_N1_K5.csv

# 2) Launch from inside the sandbox.  Absolute path to the binary;
#    log goes to the project's logs/ dir; CSVs land in
#    runs/no_wiener_N1K5/out/verify/.
env OMP_NUM_THREADS=1 OPENBLAS_NUM_THREADS=1 MKL_NUM_THREADS=1 \
nohup taskset -c 113 bash -c '
  cd /home/gyqyan/zexuan/ecg1d_cpp/runs/no_wiener_N1K5
  /home/gyqyan/zexuan/ecg1d_cpp/build/ecg1d_verify \
    --N 1 --K 5 --run-step4 \
    --T 6.283185307 --dt 1e-3 --dt-trace 1e-3 \
    --n-snap 11 --moment-form both --rt-verbose \
    --no-wiener
' > logs/realtime_N1_K5_no_wiener.log 2>&1 &
echo "PID=$!"

Monitor

# live progress

# is it still alive?

# confirm wiener=off picked up (should print "wiener=off"):

# sanity: is the OTHER job (rcond=1e-6) still alive on its own core?

# any blowups or warnings?
grep -E "BLOWUP|FATAL|WARN|drift|NaN|nan" logs/realtime_N1_K5_no_wiener.log

Success criteria (vs prior Wiener-on baselines)
- Norm should stay close to 1 across the whole period.
- Raw E*norm drift should remain bounded.
- Density / n(k) snapshots should overlay the grid reference within
  the same tolerance as the Wiener-on run; large deviations indicate
  the hard-rcond truncation is too aggressive (or not aggressive
  enough) without the smooth filter.

If --no-wiener produces NaN or visibly worse drift than the Wiener-on
baseline, the natural next steps are:
- restore Wiener filter (drop --no-wiener),
- or pair --no-wiener with --rcond 1e-6 to keep more decades.
