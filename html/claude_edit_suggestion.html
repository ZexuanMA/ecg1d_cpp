2. The HamiltonianTerms flag reminder

From src/tdvp_solver.hpp:17:

struct HamiltonianTerms {
    bool   kinetic        = true;
    bool   harmonic       = true;
    bool   delta          = true;
    bool   gaussian       = true;
    bool   kicking        = true;
    double kicking_scale  = 1.0;  // 1/T_pulse for finite-pulse kicks
};

Every term is independently toggleable. The fix is to build a pulse-only instance where only kicking is on.

5. Caveats worth knowing before you approve

  1. Trotter order. Your current loop is free(T_period − T_pulse) → kick(T_pulse), which is 1st-order Trotter. For 2nd-order (Strang) split: free(T/2) → kick(T_pulse) → free(T/2). Ask if you want this upgrade — it's a separate, small change in the for (n = 0; n < n_kicks; n++) loop body.
  2. Convergence table expectation. After Edit A, the T_pulse → 0 limit in run_kick_convergence_test should still match the exact reference (the free piece during the pulse was O(T_pulse)). For finite T_pulse (e.g. 0.5, 0.2), the numbers will shift slightly — that's expected and arguably more faithful to a pure kicked-rotor model.
  3. Gradient assembly. assemble_grad respects the flag booleans via grad_H_for_alpha, so disabling kinetic/harmonic really does remove them from g during the pulse. No hidden coupling.
  4. Scope. These two edits do not touch the ground-state SVM pipeline, run_phase3_tdvp_*, or observable code.

T1.3 — Why it diverged

The session log KICK_CONVERGENCE_SESSION.md already warned about this (before my edit):

  1. Stiffness. kicking_scale = 1/T_pulse makes the effective kick potential amplitude scale like 1/T_pulse. As T_pulse shrinks the ODE becomes arbitrarily stiff.
  2. Basis insufficiency. A 3-Gaussian basis cannot represent exp(−iκ cos(2k_L x))·ψ. Jacobi-Anger says this phase factor excites arbitrarily many momentum sectors; the bare ground-state basis has none of them.
  3. Loss of the tether (introduced by my Edit A). With kinetic+harmonic switched off during the pulse, nothing in the pulse Hamiltonian penalizes large |⟨x²⟩| or large |⟨p²⟩|. The TDVP ODE C·dz/dt = −i·g_kick can drive the Gaussian parameters arbitrarily far on a single RK4 step because g_kick is O(1/T_pulse) while C is static geometry.
Pre-edit behavior was also bad. The session log shows the convergence test was already unstable for small T_pulse before my edit — kinetic+harmonic during the pulse only delayed the divergence; they did not cure it. My edit made the problem appear earlier and more dramatically.

T1.4 — What we learned

  • The physics statement ("pulse = kick only, free = kinetic+harmonic only") is a clean Trotter split, but it requires a basis that spans the kicked subspace. With K=3 it is futile.
  • The working path in the codebase — run_kicked_gamma0_test — solves exactly this problem by (a) augmenting the basis with momentum harmonics and (b) never running TDVP on the kick. It uses analytic projection for the kick and exact propagation for the free evolution in the fixed augmented basis.
  • For Edit A/B to produce useful numbers, the test must either adopt that strategy or enlarge the basis considerably.
