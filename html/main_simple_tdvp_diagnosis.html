1. Basis function used in both reports

Both verification reports and every TDVP task in main_simple.cpp use the same explicitly-correlated-Gaussian (ECG) ansatz. For one particle (N=1) the basis function indexed by i is

\[ \phi_i(x) \;=\; u_i\,\exp\!\bigl[-(A_i + B_i)\,x^{2} + 2 R_i B_i\,x - R_i B_i R_i\bigr], \qquad \psi(x) \;=\; \sum_{i=1}^{K} \phi_i(x). \]

Each basis function carries four complex parameters, stored in the struct BasisParams (src/basis_params.hpp):

symbolrolecode field
$u_i$ linear coefficient (amplitude + phase) BasisParams::u
$A_i + B_i$ effective Gaussian width (real part) / chirp (imag part). $A$ is the variational part, $B$ is held near zero in static optimization and used to encode kicks in dynamics. A + B
$R_i$ complex center: $\mathrm{Re}\,R_i$ is the position shift, $2 B_i\,\mathrm{Im}\,R_i$ is the momentum boost. BasisParams::R

Expanding the exponent makes the shifted-Gaussian structure explicit:

\[ \phi_i(x) \;=\; u_i\, \exp\!\Bigl[-(A_i+B_i)\bigl(x - \tfrac{R_i B_i}{A_i+B_i}\bigr)^{2} + \tfrac{(R_i B_i)^{2}}{A_i+B_i} - R_i B_i R_i\Bigr]. \]

At $B_i = 0$ the linear term vanishes and $R_i$ drops out entirely — which is why the SVM initial basis is pure-$A$.

The same form generalizes to $N$ particles by promoting $A$ to an $N\times N$ symmetric complex matrix, $B$ to its diagonal, and $R$ to an $N$-vector. The TDVP variational parameter list alpha_z_list is built from these four channels via the AlphaIndex{a1, a2, a3, a4} tag with a1 ∈ {1,2,3,4} selecting (u, B, R, A).

3. Why TDVP usually fails in main_simple.cpp

Real-time TDVP solves

\[ i\,C_{\alpha\beta}\,\dot z_\beta \;=\; g_\alpha, \qquad C_{\alpha\beta} \;=\; \langle\partial_\alpha\psi|\partial_\beta\psi\rangle, \qquad g_\alpha \;=\; \langle\partial_\alpha\psi|H|\psi\rangle, \]

with $z = (u, A, B, R)$ packed into a single complex vector. The Step-4 report identifies three coupled root causes of the failure pattern observed in task5, task6, task7, task9, and task10.

3.1 Ill-conditioned $C$ + SVD truncation breaks norm conservation

Run-time inspection during Step 4 gives $\mathrm{cond}(C)\sim 3.5\times 10^{11}$ at K = 5 with $H = T + V_\text{ho}$. The pseudoinverse used in the code is

\[ C^{+}_{r} \;=\; \sum_{\sigma_i \,\geq\, r\,\sigma_1}\, \frac{1}{\sigma_i}\,|v_i\rangle\langle v_i|, \qquad r \equiv \mathrm{rcond} = 10^{-4}. \]

This drops every singular value below $r\sigma_1\sim 10^{-4}$, so the actual TDVP projector is a strict subspace $P_{T_r}\subsetneq P_T$ of the true tangent space. The resulting velocity is $|\dot\psi\rangle = -i\,P_{T_r}\,H|\psi\rangle$. The two relevant invariants evolve as

\[ \frac{d}{dt}\langle\psi|\psi\rangle \;=\; 2\,\mathrm{Im}\,\langle\psi|P_{T_r}H|\psi\rangle, \qquad \frac{d}{dt}\langle\psi|H|\psi\rangle \;=\; 2\,\mathrm{Im}\,\langle\psi|H\,P_{T_r}\,H|\psi\rangle. \]
  • Energy conservation only requires $P^\dagger = P^2 = P$ (any orthogonal projector). So $H P_{T_r} H$ is Hermitian and $\langle H\rangle$ is preserved to round-off — regardless of which subspace $P_{T_r}$ projects onto.
  • Norm conservation additionally requires $P|\psi\rangle = |\psi\rangle$. After SVD truncation this is violated and norm leaks at the rate $2\,\mathrm{Im}\,\langle\psi_T|H|\psi_\perp\rangle / \langle\psi|\psi\rangle$.

Empirically: raw K = 5 over one HO period loses 53 % of its norm, drives $|\langle x\rangle|\to 0.18$ and $|\langle p\rangle|\to 0.28$ (parity should pin both to zero), and fails to recur (fidelity drops to 0.86 at $t=2\pi$).

3.2 Why imaginary-time TDVP (task1) is fine but real-time is not

The Step-4 report's section 9.6 gives three structural reasons:

  1. Imaginary-time is steepest descent on $E$. The flow $\dot z = -C^{+}_{r}\,\nabla E$ has $\dot E = -(\nabla E)^\dagger\,C^{+}_{r}\,(\nabla E) \leq 0$ for any PSD $C^{+}_{r}$. Truncation just slows descent — it never reverses sign.
  2. Linear-coefficient resolve absorbs basis errors. With resolve_u = true (set in main_simple.cpp:282 and main_simple.cpp:551) the code re-solves $u$ exactly via the generalized eigenproblem $H_{kl} u_l = E\,S_{kl} u_l$ after every step. This soaks up any residual basis incompleteness.
  3. The minimum is internal to the manifold. Near the ground state $\nabla E$ is small and points along directions the manifold can resolve well, so truncated descent suffices.

Real-time TDVP loses all three protections. It must follow $-iH|\psi\rangle$ at every instant, so any mismatch between $T_r$ and the radial direction $|\psi\rangle$ accumulates as norm drift, and there is no analogue of resolve_u to clean it up.

3.3 More basis is not always better

Section 0 of the Step-4 report compared K = 3 vs K = 5 under the same flags:

setting relE norm_final max $|\langle x\rangle|$ verdict
K = 3 + enforce-norm + u-split, T = 0.5 6.987e-15 1.000000000 7.108e-06 best short test
K = 5 + enforce-norm + u-split, T = 0.5 3.003e-02 1.000000000 3.676e-01 reject
K = 3 split-norm, T = 2π 1.780e-12 1.000000000 8.282e-03 full recurrence passed

Adding more basis functions at fixed conditioning makes the C-matrix more degenerate (similar Gaussians span nearly the same direction), pushing more singular values below the rcond cutoff. The truncated subspace then leaves out more of the radial mode rather than less. This means the natural “just add more K” reflex is wrong for real-time evolution.

Notably, prepare_kicked_state_route1 in main_simple.cpp:1004-1053 can grow K via augment_basis_with_momentum (the n_mom parameter adds momentum copies of the pre-kick basis), so even with --K-cap 3 a Route-1 kick may push K past 3 unless the cap is enforced.

4. Practical knobs to flip when a real-time task fails

In order of importance:

  1. Cap K to 3. Pass --K-cap 3 (handled at main_simple.cpp:2236) so that the C-matrix degeneracy stays manageable.
  2. Keep --enforce-norm on. This rescales $u\to\sqrt{n_0/n_t}\,u$ each step to clamp $\langle\psi|\psi\rangle = 1$. It does not fix the trajectory shape, but it isolates the norm leak from the rest of the dynamics.
  3. Keep --u-split-trotter on, and remove $u$ from alpha_z_list. In split-Trotter mode TDVP only updates the geometry $\{A, B, R\}$, and $u$ is propagated analytically by $\exp(-iH\,dt)$ on the updated basis. task5 at main_simple.cpp:1100-1102 already excludes $u$ when u_split=true; task6 at main_simple.cpp:1226-1230 and task10 keep $u$ in the list, which is incompatible with u_split_trotter and is a likely failure mode.
  4. Loosen rcond only as last resort. A larger truncation threshold can amplify numerical noise. Section 10 of the Step-4 report rates this option D — risky.

Takeaway. Energy conservation is necessary but not sufficient for real-time TDVP — it follows from $P_{T_r}$ being an orthogonal projector and tells you nothing about which subspace was projected out. Always check norm, parity-protected observables ($\langle x\rangle, \langle p\rangle$), and recurrence. The currently validated main_simple.cpp recipe for an N = 1 real-time run is K = 3 with both --enforce-norm and --u-split-trotter enabled, and $u$ excluded from the TDVP parameter list.
