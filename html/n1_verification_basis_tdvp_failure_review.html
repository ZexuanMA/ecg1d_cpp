N=1 Verification Basis and TDVP Failure Review

Generated 2026-04-27. Scope: review of out/verify/N=1_verification.html, out/verify/N=1_step4_verification.html, and main_simple.cpp.

Short conclusion. The verification uses a complex one-dimensional ECG basis psi = sum_i u_i phi_i. For N=1, the verification avoids the generic SVM path and manually seeds HO ground state + randoman perturbations. main_simple.cpp often fails because its Task A still calls svm_build_basis(N=1, K>1), but that path inserts random_basis_2particle() objects hardcoded to N=2. Raw real-time TDVP has a separate failure mode: the tangent metric C is ill-conditioned, so SVD regularization preserves energy while leaking norm and breaking observables.

1. Basis Function Used in Verification

For one particle, the ECG ansatz used by the verification is:

psi(x) = sum_i u_i phi_i(x)
phi_i(x) = exp[-(A_i + B_i) x^2 + 2 R_i B_i x - R_i B_i R_i]

All four parameters are complex numbers stored in BasisParams. For N=1, A and B are 1 by 1 matrices and R is a one-entry vector. The effective width/chirp is A_i + B_i. When B = 0, the linear term vanishes and R drops out of the basis function.

SymbolCode fieldMeaning for N=1
u_iBasisParams.uLinear coefficient, amplitude and phase.
A_i + B_iBasisParams.A + BasisParams.BGaussian width from real part, chirp from imaginary part.
R_iBasisParams.RComplex center. Its real part shifts position; with nonzero B, imaginary part carries momentum phase.

2. Verification Setup

Step 1: Ground State

For N=1, the verification prepares the ground state of:

H_full = T + V_ho + kappa cos(2 k_L x)

It uses K=5 basis functions, initialized as one harmonic-oscillator ground-state Gaussian plus four randoman perturbations. This is intentionally different from main_simple.cpp Task A. 

// src/verify/step1_imag_time_ecg.cpp
if (N == 1) {
    basis.push_back(HO_ground_state_A_0_5_B_0_R_0);
    for (int k = 1; k < K; k++) {
        basis.push_back(BasisParams::randoman(N, 42 + k, k));
    }
}

Step 4: Real-Time Verification

Step 4 starts from the Step 1 ECG ground state of H_full, turns off the cosine, and evolves under:

H_evolve = T + V_ho = -1/2 d^2/dx^2 + 1/2 x^2

For exact harmonic evolution over T_total = 2 pi, the state should recur. The reference grid and DVR solvers diagonalize H_evolve and propagate spectrally. Their agreement is the truth baseline.

3. Why main_simple.cpp Usually Fails

Failure 1: Task A Uses the Wrong N=1 SVM Path

In main_simple.cpp, Task A calls: 

SvmResult svm = svm_build_basis(N, K, 2000,
                                HamiltonianTerms::kinetic_harmonic(),
                                42, 0.0);

This is unsafe for N=1, K>1. Inside svm_build_basis, every added trial basis comes from random_basis_2particle, which is hardcoded as: 

BasisParams random_basis_2particle(std::mt19937_64& rng, int name) {
    int N = 2;
    ...
}

The verification source explicitly avoids this for N=1: 

// svm_build_basis / stochastic_refine both use random_basis_2particle
// internally, which is hardcoded for N=2 and produces invalid N=1
// trial basis members.
// Therefore seed manually for N=1.
Reproduced command: ./build/ecg1d_simple --task1 --N 1 --K 5. It generated impossible harmonic-oscillator energies below 0.5, then exited with code 139. That is not a physical TDVP failure. It is an invalid mixed-dimension basis construction path.

Failure 2: K=1 Is Already Exact, but the Line Search Requires Strict Decrease

For the pure harmonic oscillator, one ECG with A = 0.5, B = 0, R = 0 is already the exact ground state. Running ./build/ecg1d_simple --task1 --N 1 --K 1 gives the correct final energy 0.5. However, the TDVP loop prints many line-search failures because tdvp_step only accepts a trial step if: 

if (E_eigen < E_pre.real()) {
    return success;
}

At the exact minimum, the correct update is zero and the energy cannot decrease. The code reports a line-search failure even though the result is already converged.

This is a convergence-logic issue. It should stop cleanly when |dz| is small or when |E_new - E_pre| is below tolerance, instead of requiring strict decrease.

Failure 3: Raw Real-Time TDVP Is Ill-Conditioned

The Step 4 report shows that raw real-time TDVP can conserve energy while still giving the wrong trajectory. The raw K=5 run had:

The diagnostic cause is the tangent-space Gram matrix C. In the report, the raw K=5 real-time run has cond(C) approximately 3.5e11 to 4.0e11. The SVD cutoff rcond = 1e-4 drops motion components. The remaining projected flow is still energy-conserving, but it is no longer norm-preserving and no longer parity-stable.

The accepted Step 4 setting after the follow-up checks is: 

./build/ecg1d_verify --N 1 --K 3 --run-step1 --run-step4 \
  --enforce-norm --u-split-trotter
ControlMeaning
--enforce-norm After each real-time step, rescale all u coefficients to restore the initial norm.
--u-split-trotter Remove u from the TDVP alpha list. TDVP updates nonlinear geometry first, then fixed-basis propagation evolves u.
K=3 Less overparameterized than raw K=5 for this recurrence test. The report rejected K=5 with split-u in the short test.

4. Practical Fixes for main_simple.cpp

  1. For N=1, do not call svm_build_basis with K>1 until the SVM sampler is generalized. Copy the verification initializer: one HO Gaussian plus BasisParams::randoman(1, seed, name) perturbations.
  2. For pure HO ground-state teaching examples, use K=1 and skip TDVP polish when the initial energy is already within tolerance of 0.5 * N.
  3. Relax TDVP line-search success at stationary points. Accept convergence when |dz| < tol or |dE| < energy_tol, even if there is no strict decrease.
  4. For real-time N=1 evolution, prefer the verification controls: K=3, enforce_norm=true, and u_split_trotter=true.
  5. Treat energy conservation alone as insufficient. Always inspect norm, <x>, <p>, density fidelity, and recurrence fidelity for real-time TDVP.
