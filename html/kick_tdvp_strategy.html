Part 1 — 诊断：为什么现在不行

1.1 物理背景与当前配置

要模拟的是 Lieb-Liniger 量子踢转子（QKR）：

H(t) = ∑a[ −½ ∂xa² + ½ ω² xa² ] + g ∑a<b δ(xa−xb) + κ ∑a cos(2kLxa) · ∑n δ(t − nT)

你的代码里把这个 Floquet 演化做 Trotter 分裂：

U(T) ≈ Ufree(T − Tpulse) · Ukick(Tpulse)
Ukick(Tpulse) = exp[−i · (κ/Tpulse) ∑a cos(2kLxa) · Tpulse] = exp[−i κ ∑a cos(2kLxa)]

所以 pulse 窗口的 "kicking_scale = 1/T_pulse" 是在模拟 Tpulse → 0 极限下的瞬时 δ-kick。现在的 run_kick_convergence_test 在 pulse 窗口用 RK4 + TDVP 积这个高幅值短脉冲。

1.3 发散的三个协同原因（按严重程度排列）

原因一 · 基底表达能力不足（最根本）

Kick 算符通过 Jacobi–Anger 展开作用到波函数上：

e−iκ cos(2kLx) = ∑n=−∞+∞ (−i)n Jn(κ) · ei·2nkLx

也就是说，一次 kick 把波函数同时乘上无穷多个动量平面波 ei·2nkLx，权重由 Bessel 函数 Jn(κ) 给出。当 κ=1 时权重衰减得还算快：

J0(1)=0.765, J1(1)=0.440, J2(1)=0.115, J3(1)=0.020, J4(1)=0.0025

即使截断到 |n|≤4，也需要至少 9 个不同动量扇区的基态。但你的 K=3 基底只有三个 R=0 的实高斯函数——这三个高斯只能构造 中心在零动量附近 的态：

ψbase(x) = ∑i=13 ui · e−Aix² ⟹ ⟨p⟩ = 0, ⟨p²⟩ ≲ max(Ai)

一次 κ=1 的 kick 产生的真实态里有至少 30% 的概率在 |p| ≥ 2kL 的动量扇区。K=3 基底表示这个态的投影保真度 ≪ 1。

这是 "流形不兼容" 问题。 TDVP 在变分流形 M 上演化，但 Ukick|ψ⟩ ∉ M。TDVP 只能返回 M 内对 Ukick|ψ⟩ 的最佳近似（关于某个度量）。如果 M 根本不含被踢态的主要成分，这个最佳近似和真实态差得没边。

原因二 · 短脉冲的刚性（stiffness）

为了让有限脉冲极限逼近理想 δ-kick，代码让 kicking_scale = 1/T_pulse。这导致 pulse 窗口的哈密顿量：

Hpulse = (κ/Tpulse) ∑a cos(2kLxa)

幅值随 Tpulse → 0 发散。TDVP 方程是

C(z) · ż = −i · g(z), 其中 gα = ∂⟨H⟩/∂zα*

当 H ∝ 1/Tpulse，g ∝ 1/Tpulse，而 C 只是基函数几何的 Gram 矩阵（与 Tpulse 无关）。所以

ż ∝ 1/Tpulse

RK4 的稳定性要求 Δt · ‖ż‖ ≲ O(1)，即需要 Δt ≲ Tpulse。代码里 dt_kick = min(T_pulse/50, 0.05/(scale·κ)) = min(T_pulse/50, 0.05 T_pulse/κ)，本意就是保证这一点。

但这只保证了 "每步 Δz 是小量"，没有保证 "整个 pulse 的累积 Δz 是小量"——累积量 ∫0Tpulse ‖ż‖ dt ∝ (1/Tpulse) · Tpulse = O(1)，与 Tpulse 无关。问题就是这个 O(1) 的位移往哪里去——原因一说了，它往流形外面去。

原因三 · 失去回拉力（加速了原因一、二的灾难）

按 Edit A 的"纯 Trotter"策略，pulse 窗口只留 kicking 项，把 kinetic、harmonic 全关掉。此时 pulse 窗口的 TDVP 方程里：

  • 只有一个 cos(2kLx) 型势能
  • 没有任何项惩罚 "高斯变宽" / "中心漂移" / "动能爆炸"

具体地，g = ∂⟨ψ|cos(2kLx)|ψ⟩/∂A* 对高斯宽度参数 A 的梯度并不小，而且 没有对偶项把 A 拉回来。一旦 RK4 把 A 推到负的实部，基函数 e−Ax² 爆炸，后续计算就失效。

原因一是必要条件，原因二、三是放大器。 即使你解决了原因二（把 Tpulse 设成 O(1)，让刚性消失）、三（保留 kinetic+harmonic），只要基底没有动量谐波，TDVP 也只是"慢性中毒"——前几步看起来稳、几十个 kick 后一定崩。

Part 2 — ECG 基底 × Kick 算符：结构性冲突

2.1 ECG 基底能表达什么

单个 ECG 基函数：

φ(x) = exp[−xT(A+B)x + 2RTBx − RTBR]

关键参数：

  • A：实对称正定（通常），决定高斯宽度和粒子间关联（off-diagonal）。
  • B：对角实矩阵，与 R 一起定义中心位置。
  • R：N 维复向量。实部是高斯中心（实空间位移），虚部是动量相位 ei·2B·Im(R)·x。
  • u：线性系数（CI-style）。
关键事实 1：一个 R 虚部非零的 ECG 就是一个"动量谐波 × 高斯"。 把 R = Rr + i·Ri 代入：
φ(x) = e−(x−Rr)T(A+B)(x−Rr) + 2i RiTB(x−Rr) · (const)
这是中心在 Rr、动量为 2B·Ri 的高斯波包。
关键事实 2（最重要）：kick 算符作用在 ECG 上仍然是 ECG 的线性组合。
ei·2nkLx · e−(A+B)x² + 2RTBx − RTBR = e−(A+B)x² + 2(RTB + i·nkLeT)x − RTBR
也就是 同样的 A、B，R 被替换成 R' = R + i·nkL·B⁻¹·e。 这说明 ECG 流形 在 "乘以动量平面波" 这一操作下封闭——而 Ukick 正是这些平面波的 Bessel 加权叠加。

2.2 为什么一个 ECG 不够、但"扩大的 ECG 流形"够

一次 kick 后：

Ukick|φR⟩ = ∑n=−∞+∞ (−i)n Jn(κ) |φR + inkₗB⁻¹⟩

这是对 无穷多 个 ECG 的加权求和。但两件事救场：

  1. Bessel 截断。 当 n 大于某个阈值（取决于 κ），Jn(κ) 指数衰减。κ=1 时 |n|≤4 足够；κ=10 时大概 |n|≤15。实用上有限维就够。
  2. 高斯 damping 因子。 代码里 single_particle_kick_kernel 里的 exp(−n²kL²·Kinv(a,a)) 表明重叠 ⟨φR|φR+inkₗ⟩ 高斯衰减；基底两两间的"远距离"动量扇区耦合指数小。
所以 ECG 和 kick 的关系不是"根本不相容"，而是"单个 ECG 不够、但覆盖 |n|≲nmax 动量扇区的 ECG 集合刚好够"。 这正是 augment_basis_with_momentum 和 --kicked-gamma0 路径的哲学。

2.3 ECG 相对于其他基底的优劣

基底优势劣势（对 kick）
ECG（你现在用的）
  • 所有算符矩阵元 闭式解析（高斯积分）
  • 粒子间关联通过 off-diagonal A 自然编码，不需要 CI 展开
  • 对称化通过 N! permutation sum，规模可控到 N≲6
  • 在 R 虚部自由度下已经能表达任意动量高斯
  • 一个 ECG 只覆盖 一个 动量扇区
  • 动量多扇区必须通过 K 个基函数覆盖 → 基底尺寸线性膨胀
  • 非正交基，S 矩阵条件数可能很坏
平面波（Fourier）
  • kick 是 对角矩阵（只移动 n → n+m）
  • 动能也是对角
  • 可以做 split-step FFT
  • delta 相互作用难处理（需要正则化）
  • 粒子数 N 多时指数膨胀
  • 空间局域化（如 harmonic trap 基态）需要大量 k 点
谐振子本征基
  • free evolution 是对角相位
  • matrix elements 已知
  • kick matrix element 需要 Hermite 积分，涉及 Hermite-Gauss 正交
  • N 粒子多体反对称化复杂
  • delta 同样麻烦
MPS / DMRG
  • 纠缠结构可调、bond dim 控制精度
  • 对 1D 局部哈密顿量高效
  • 连续空间要先离散化（lattice）
  • long-range delta 在连续空间里没有自然 MPO 表示
  • 完全不同的代码基础

结论：ECG 对 kick 不是最好（平面波最好），但对 "kick + delta + harmonic" 联合是 最平衡 的选择。关键是要把 R 虚部自由度用起来。

Part 3 — 改进策略（7 条，逐个点评）

下面 7 条按"改动量从小到大、物理代价从低到高"排列。每条给出：数学原理、预期稳定性、实现成本、物理代价。

A. 解析 kick + TDVP free（已部分实现）
实现：极易 稳定性：高 物理代价：中

思路

完全绕开"pulse 窗口 TDVP"。pulse 视为瞬时，用 apply_analytic_kick 完成；free 窗口用 realtime_tdvp_step（或者固定基的 free_evolve_fixed_basis）。

数学原理

利用 Jacobi-Anger 解析展开，直接在基底上构造 kick 矩阵元：

Kij = ⟨φi|e−iκ∑acos(2kLxa)|φj⟩ = ∑n1...nN ∏a[(−i)naJna(κ)] · ⟨φi|ei·2∑anakLxa|φj⟩

每个 ⟨φi|ei·2nkLxa|φj⟩ 是高斯积分，有闭式。这就是 src/kick_operator.cpp:18-51 的 single_particle_kick_kernel。

然后通过广义线性代数 u' = S⁻¹ K u 把 kick 作用到线性系数 u 上（不动 A、B、R 参数）。

优点

  • Kick 本身 精确（到 Jacobi 截断阶 n_bessel）
  • Kick fidelity 可量化，直接告诉你基底够不够
  • 没有刚性问题（不做 ODE 积分）
  • 代码已经完整，只差在 run_kick_convergence_test 里调用

缺点

  • 基底必须预先扩大。 如果基底只 K=3，fidelity 会很低（~0.85），误差直接从第一次 kick 就固定下来。
  • 无法回答"有限脉冲 TDVP 是否收敛到理想 kick"这个学术问题——它 跳过 了这个问题。
  • 每次 kick 之间基函数不变，但 u 可能把 weight 集中到少数基函数 → 长时间 norm drift 可能累积。

实施难度：

5 分钟。把 run_kick_convergence_test 里 apply_kick 的 TDVP 分支替换成 apply_analytic_kick。已有的 run_kicked_gamma0_test 就是这样做的。

推荐用法

作为 生产路径，尤其是想算 MBDL 长时间（n_kicks ≥ 100）动力学时。Kick 误差被 基底 控制，不被 时间积分 控制，这对数百次 kick 的累积稳定性至关重要。

B. 预先动量增广 + TDVP pulse
实现：易 稳定性：中 物理代价：低

思路

在进入 kick 循环之前调用 augment_basis_with_momentum(basis, kL, n_mom=4, b_val=0.5)，把基底从 K=3 扩到 K~30+。然后 pulse 窗口仍然走 RK4-TDVP，但此时 流形已经覆盖了主要动量扇区。

数学原理

增广后基底包含 {φR=inkₗ/b}，n ∈ [−nmom, nmom]。一个 Bessel 展开截断到 |n|≤nmom 的 kicked 态就落在这个扩展流形里。

pulse 窗口 TDVP 方程仍然成立，但 ‖ψ − PMUkickψ‖ 现在是 Bessel 尾巴 ~Jnmom+1(κ) 而非 O(1)。

优点

  • 保留"TDVP 全过程"的学术正确性（测的是真·有限脉冲极限）
  • 基底够大时，RK4 能跑
  • augment_basis_with_momentum 已经实现

缺点

  • K 从 3 涨到 ~30，单步 TDVP 代价涨 100×（C 是 O(K·d)²，d 是参数数）
  • S 条件数变坏（momentum 高斯之间重叠非零），需要调 max_cond
  • 刚性问题 没解决：Tpulse 太小 RK4 还是崩，只是崩得慢一点
  • 实测（Trial 1 边注）：光增广到 K=15 对 T_pulse=0.1 仍然不稳

实施难度：

15 分钟。在 run_kick_convergence_test 里 SVM 完之后、循环之前插一行 augment_basis_with_momentum。但要重新确认 alpha_z_list（因为 K 变了）。

物理代价

低——基底增广本身是合法的变分扩张，不改哈密顿量。

推荐用法

作为 学术验证路径——如果你 必须 回答"有限脉冲 TDVP 收敛到 δ-kick"，跟 A 搭配使用（A 做生产，B 做验证）。

C. 解析 boost — 在每次 kick 把 R 虚部移一下
实现：中 稳定性：高 物理代价：低（有截断）

思路（novel，我自己想的）

不建立 kick 矩阵，也不做 TDVP 积分。每次 kick 时，对基底每个函数执行 Jacobi-Anger 展开：把一个基函数 φR 替换成 |n|≤nmax 的加权和 ∑n(−i)nJn(κ)φR + inkₗB⁻¹。这会让基底尺寸 K → K·(2nmax+1)。然后 定期裁剪（把小权重的基函数合并或丢弃）。

数学原理

e−iκ cos(2kLx) · φR(x) = ∑n(−i)nJn(κ) · φR + inkₗB⁻¹(x)

因为 R 虚部平移正好是"乘动量平面波"操作，所以 Ukick 作用在基底上是 每个基函数派生出 2nmax+1 个"儿子"，儿子就是原函数 R 虚部偏移了 ±kL/b, ±2kL/b, ... 的版本。这是严格的，不涉及任何近似（除了 nmax 截断）。

优点

  • Kick 是 严格解析 作用（到截断阶），不做 ODE 积分，完全没刚性
  • 与 A 相比，u 不需要重解（因为基底本身已经被 kick 过了）
  • 与 B 相比，自适应——只在真正被 kick 的方向上增广，不做预先大而全的增广
  • 每次 free 演化的基底 主动跟踪 波函数的动量扇区分布

缺点

  • 基底尺寸指数增长——每次 kick 乘 (2n_max+1)，n_kicks=100 时不可控
  • 必须配合 基底压缩：比如 SVD 截断、CUR-decomposition、或合并几乎共线的基函数
  • 实现复杂度比 A/B 高一个量级
  • 压缩引入误差，可能在 n_kicks 多时慢慢积累

实施难度：

2-3 天。需要：(1) 实现 R-virtual-shift 的 kick 作用，(2) 实现基底压缩器（SVD on overlap matrix + 重新展开），(3) 调 n_max 和压缩阈值。

数学细节

压缩的关键：在新基底 {φj} 上算 Gram S，SVD 得到 S = V D V†。截断 D 的小奇异值，保留 V 的前 K' 列作为"有效基"。然后把波函数投影到这个子空间。这保证压缩误差可量化（=被扔掉的奇异值之和）。

推荐用法

如果 Strategy B 的基底尺寸压力太大（比如你想做 N=4，K×(2nmom+1)N 会爆），C 是更 scalable 的路线。对 N=2,3 没必要，直接用 A+B。

D. 隐式 / 指数积分器替换 RK4
实现：难 稳定性：看情况 物理代价：低

思路

把 realtime_tdvp_step 里的 RK4 换成隐式中点法 (implicit midpoint) 或 Gauss-Legendre 隐式 Runge-Kutta，或者 exponential integrator（把线性部分 exp 精确处理）。

数学原理

隐式中点法：

zn+1 = zn + Δt · f((zn+zn+1)/2)

这是 辛 且 A-stable 的，对任意刚性问题都稳定（误差可能大，但不会爆炸）。代价是每步要解一个非线性方程（Newton 迭代）。

Exponential integrator：把 ż = Lz + N(z) 的线性部分 L 用 exp(LΔt) 精确处理：

zn+1 = eLΔtzn + Δt·φ1(LΔt)·N(zn)

其中 φ1(z) = (ez−1)/z。这对"大振幅但线性"的 kick 特别合适——cos 作为位置函数，在固定基底下 ⟨φi|cos(2kLx)|φj⟩ 是常数矩阵，可以看成"线性"的。

优点

  • 彻底消除刚性问题
  • 对任意 Tpulse 都能跑

缺点

  • 隐式法每步要 Newton 迭代 → 性能 3-5× 下降
  • Newton 的 Jacobian 涉及 TDVP 方程的二阶导，代码量爆炸
  • 不能解决基底不够的问题——积分器再好，流形投影丢的信息也找不回来
  • Exponential integrator 要求 L 是常数矩阵，但 TDVP 里 C 依赖 z，不是常数

实施难度：

1-2 周。Eigen 不自带隐式 RK，需要自己写 Newton 迭代。要验证辛性、能量守恒。

推荐用法

不推荐单独用——不解决根因。除非你已经搞定基底（用 B 或 C），然后想把 Tpulse 推到极小以做"理想 δ-kick" 的学术验证，此时隐式法是最后一公里的事。

E. Magnus 展开 —— 把 pulse 整窗解析化
实现：难 稳定性：高 物理代价：可量化

思路

对 pulse 窗口整个 Tpulse，不做 RK4 离散，而是写 Upulse = exp(Ω)，Ω 由 Magnus 级数给出。一阶截断就是：

Ω1 = −i ∫0Tpulse H(t) dt = −iκ cos(2kLx) (吸收了 1/Tpulse × Tpulse)

二阶：

Ω2 = −½ ∫0Tpulsedt1∫0t1dt2[H(t1),H(t2)]

对理想 δ-kick，H(t) 与自身对易，Magnus 级数在一阶截断——这就是解析 kick。对有限 Tpulse + 同时有 kinetic/harmonic 的情形，[T, cos(2kLx)] ≠ 0，Magnus 二阶给出 Tpulse² 修正。

优点

  • 不需要 ODE 积分，整个 pulse 一次解析
  • 在基底 有限维 下，Ω 是 K×K 矩阵，exp(Ω) 用 Padé 或谱分解精确算
  • 有限 Tpulse 的修正可量化（= Magnus 截断误差）
  • 辛（严格酉）

缺点

  • Magnus 系数涉及嵌套对易子 [H(t1),H(t2)]，解析计算到二阶以上非常痛苦
  • 对于 δ-kick 极限，Magnus 一阶 等价于 Strategy A，所以收益有限
  • 对 Tpulse 有限的情况，二阶修正需要实现 [T+V, cos] 的基函数矩阵元，工作量大

实施难度：

2-3 周。核心工作是把对易子矩阵元解析化。

推荐用法

如果你想把 Strategy A 推广到 有限 Tpulse（比如模拟实验中的 20μs 脉宽）并保持解析性，Magnus 是正确路径。否则 A 已经够了。

F. Strang 分裂 + 动态基底再增广
实现：中 稳定性：高 物理代价：低

思路

把每个 kick 周期分解成：

U(T) ≈ Ufree(T/2) · Ukick(解析) · Ufree(T/2)

这是 Strang splitting，精度 O(T³)，比 1 阶 Trotter 的 O(T²) 高一阶。free 段用 TDVP (或固定基 exact)，kick 段用解析 (Strategy A 或 C)。

关键加料：free 段结束后，检查基底对当前波函数的表达保真度；如果 fidelity 明显掉了（说明 free 演化把波函数推到了流形外），做一次 在线基底扩张（把当前波函数的 "residual" 加入基底）。

数学原理

Strang 误差项是嵌套对易子 [[Hfree, Hkick], ...]·T³/24。对实验参数 (T=1, κ=1)，这个误差 ~10⁻²，比 1 阶 Trotter (~10⁻¹) 好一个量级。

在线扩张：定义 residual |r⟩ = |ψ⟩ − PM|ψ⟩。把 |r⟩ 拟合成新 ECG（最小二乘），加到基底里。

优点

  • 物理上正确的 Trotter 展开
  • 动态基底：只在真正需要时扩张，控制 K 膨胀
  • 完美结合 A 和 C 的优点

缺点

  • 在线拟合新 ECG（把 |r⟩ 近似成 ∑ ci·ECGi）是非线性最小二乘，本身不简单
  • 基底再优化时 S 条件数要重新监控
  • 如果 fidelity 下降太快，可能要每步都扩张 → 退化成 Strategy C

实施难度：

1 周。主要工作在在线拟合模块。

推荐用法

中长时间（n_kicks ~ 100）动力学的"软工程"方案。在 A 的基础上加 Strang 分裂即可快速上手，在线扩张是锦上添花。

G. 流形扩展：把 R_im、A_off-diag 全放进 TDVP 变量
实现：中 稳定性：待验证 物理代价：低

思路

你的 SolverConfig::dynamics() 已经打开 optimize_B 和 optimize_R，但默认 optimize_A 也只含 diagonal。对 kick 来说，最相关的自由度是 R 的虚部（动量），因为 kick 直接把 R 虚部移位。如果 TDVP 里包含 R 虚部作为主动变量，那么即使基底只有 K=3，每个基函数也能"跟着 kick 飘动"到正确的动量位置。

数学原理

让 TDVP 变分流形是：

M = { ∑i ui·exp[−(Ai+B)x²+2RiTBx−...] : u, A, R 都复 }

变分参数数：K × (1 + N(N+1)/2 + N) 复自由度。对 N=2, K=3：3×(1+3+2)=18 复 = 36 实自由度。

kick 算符作用后，"最佳拟合"是把 Ri → Ri + i·⟨J⟩·kL/B（某种 Bessel 加权平均）。TDVP 方程会自动把 R 推向这个方向，只要积分器能稳定处理。

优点

  • 基底保持最小 K，计算量最低
  • 物理上优雅：让基函数"随波逐流"
  • 单个高斯波包就能跟踪一个动量的经典轨迹

缺点

  • kick 产生的是 多动量叠加（Bessel 权重覆盖 |n|≤4），单个 R 只能跟一个动量 → 信息仍然丢
  • K=3 基底里每个基函数的 R 都往"平均动量"飘，不能分别覆盖 n=1, n=2, n=3 扇区
  • 变分流形几何变坏（大量零模），C 矩阵更病态

实施难度：

已经存在！把 config = SolverConfig::dynamics() 打开即可。测一下就知道行不行。

数学上的根本限制

这个策略假设"每个基函数跟踪一个经典轨迹"的 TDVP 半经典近似。对 κ≪1 弱 kick 或许可以，对 κ~1 中等 kick 肯定不够，因为 kick 不是位移、是多扇区展开。单高斯包 ansatz 的经典极限丢掉量子干涉。

推荐用法

作为 快速试水：打开 dynamics 模式跑一次，看 fidelity 如何。如果有意外惊喜（可能性小）就用；大概率只是证明"单高斯包覆盖多动量扇区"是不可能的，从而佐证 B/C 的必要性。

Q1 · δ(t) 里的 1/T_pulse 到底在说什么？kick 应该表达成什么？用 delta 合理吗？
原问题。"为什么要让 kicking_scale = 1/T_pulse，脉冲不就是把 delta 函数换成一个很小的数字嘛，等一下这个哈密顿量是啥意思，这个外部注入 kick 应该怎么表达呀，因为这个你可以看作一种力，他最后要转化成系统的势能，或者某一种能量，用 delta 合理吗"

A1.1 先看清 δ 是在时间还是空间里

Paper 和你代码里的哈密顿量其实长这样：

H(t) = T + Vho + g·δ(xa−xb) + κ·cos(2kLx)·∑nδ(t−nT)

这里有 两个 δ 函数，含义完全不同：

  • δ(x1−x2) — Lieb-Liniger 的 空间接触相互作用，代表两粒子靠在一起时的 s-wave 散射。这是 CLAUDE.md 里的 "delta contact interaction"，跟 kick 没关系。
  • δ(t−nT) — QKR 的 时间瞬时 kick，代表在 t=nT 的瞬间给整个云打一发光晶格脉冲。

"用 delta 合理吗"讨论的是 后者（时间 δ）。不要跟空间 δ 混淆。

A1.2 为什么 kicking_scale = 1/T_pulse

时间 δ 函数的物理意义是"把一次脉冲在无限短的时间里注入完"。理想 kick 作用一次产生的幺正演化是：

Ukick = exp[−i∫nT−nT+ Hkick(t) dt] = exp[−iκ·cos(2kLx)]

用到了 δ(t) 在邻域内积分等于 1。关键是 κ·cos(2kLx) 本身就是那个时间积分的结果，而不是瞬时幅值。

实验上你做不到真正的 δ(t)，所以把 δ(t) 换成"宽度 Tpulse、面积为 1 的矩形"：

δ(t) → χ[0, Tpulse](t) / Tpulse

代到哈密顿量里，脉冲期间的势就是 (κ/Tpulse)·cos(2kLx)。这个 1/Tpulse 不是玄学——它 保证脉冲的时间积分面积不变：

∫0Tpulse (κ/Tpulse)·cos(2kLx) dt = κ·cos(2kLx)
关键点。 如果你 不 加 1/Tpulse，那 Tpulse → 0 时整个脉冲的积分也 → 0，根本没 kick。1/T_pulse 出现的目的是让"有限脉冲"在 Tpulse → 0 极限下退回到理想 δ-kick。

A1.3 "kick 是力，会转成什么能量"——这步直觉有点偏

Kick 本身 不是力，而是 时间依赖的势能项（力是 −∂V/∂x；量子力学里我们直接写 H 不写 F）。能量转移链是这样的：

  1. 脉冲开 → 势能 Vkick(x,t) 存在
  2. 此势能对波函数做 位置依赖的相位旋转 exp[−iκ·cos(2kLx)]
  3. 脉冲关 → Vkick 消失，但波函数已经带上了空间依赖的相位
  4. 这个相位在后续自由演化中表现为 动能（p̂ 作用在 eiφ(x) 上 = ∇φ，所以空间相位 = 动量分布）
所以注入的能量最终变成动能，不是势能（脉冲一结束 Vkick 就不存在了）。这正是 MBDL 要研究的核心——没有 localization，每次 kick 动能累加发散；有了 localization，动能饱和。

A1.4 δ(t) 在三个层面是否合理？

层面是否合理理由
数学✅ 合理 δ(t) 定义良好，exp[−iκ·cos(2kLx)] 是标准幺正算子；所有经典教材这样写 QKR。
实验✅ 合理 Guo 等人：Tpulse ≈ 300 ns、T ≈ 50 μs，比值 ~0.6%，确实接近理想 δ。
数值⚠ 有陷阱 合理的前提是你 解析地利用 1/Tpulse 与 Tpulse 的抵消（Jacobi-Anger），而不是让 RK4 自己发现它。

A1.5 数值陷阱的具体原因

在解析层面：

(κ/Tpulse) · Tpulse = κ     [精确]

在 RK4 层面：

势能 ~ 1/Tpulse（很大）× 步长 ~ Tpulse/50（很小） = O(1) 的"每步相位"

看起来似乎也是 O(1)，但 RK4 只会看到瞬时梯度大，不知道这个大梯度会被短时间抵消。它在每一步 Δz = ż·Δt 的局部计算里把 ż ∝ 1/Tpulse 完整吃进去，再乘一个小 Δt，误差项（RK4 是 O(Δt5) 局部截断）里带了 ż5 ∝ 1/Tpulse5 的系数——这个系数不与 Tpulse 在分子上的 Δt5 精确抵消，所以 Tpulse → 0 时误差发散。

这就是 Strategy A（解析 kick）存在的根本原因：它直接在代数层面完成那个抵消，把 exp[−iκ·cos(2kLx)] 作为一个 Jacobi-Anger 矩阵元算出来，不让积分器碰这个大数。

A1.6 如果想"更物理"地表达脉冲

标准几种选择，按逼近程度排列：

脉冲形状哈密顿量→δ 极限数值特点
理想 δ κ·cos(2kLx)·δ(t) 本身就是 必须解析处理（Jacobi-Anger）
矩形脉冲 (κ/Tp)·cos·χ[0,Tp](t) Tp→0 收敛 阶梯状，Magnus 收敛好，但 1/Tp 刚性
高斯脉冲 (κ/√(2π)σ)·cos·e−t²/2σ² σ→0 收敛 光滑，高阶数值积分适用，但峰值仍 ~1/σ
Sech 脉冲 (κ/πτ)·cos·sech²(t/τ) τ→0 收敛 量子光学常用，解析可积

但 关键一点：只要你想研究 MBDL，脉冲形状几乎不影响结论——因为 Floquet 动力学只看"每周期积分到的算符"（首阶 Magnus 等价于 Ukick = exp[−iκ·cos(2kLx)]，与脉冲形状无关）。形状只在 Magnus 二阶及以上有修正。所以在物理上，δ(t) 就是 MBDL 问题的正则模型，没有必要换成别的形状，除非要专门研究"有限脉冲宽度修正"。

A1.7 用你的话总结

"脉冲就是把 delta 函数换成一个很小的数字"——不完全对。

更准确的说法是："脉冲是把 δ 函数换成一个 宽度很小、幅值很大、面积保持 κ 不变 的函数"。宽度和幅值是反比关系：Tpulse × (κ/Tpulse) = κ。如果只把宽度变小而不放大幅值，脉冲面积就变成 0，什么 kick 也没打。

"kick 是力、会转成势能"——也不完全对。 Kick 是时间依赖的 势能算符，作用方式是乘相位 exp[−iκ·cos(2kLx)]。相位一旦乘上，就作为 空间依赖相位 储存在波函数里，脉冲结束后 Vkick 消失，这个相位表现为 动量分布变化，即转化成了 动能。势能只是在脉冲瞬间作为"中介"存在。

"用 δ 合理吗"——合理，但要用对地方。 物理上合理（标准模型）、实验上合理（Tpulse/T ~ 1%）、数学上合理（幺正算子定义良好）；但数值上要求你用解析方法处理 kick（Strategy A），不能让 RK4 硬积。
Q2 · 这个哈密顿量定义本身是否合理？能被薛定谔方程正确推动吗？
原问题。"定义哈密顿量最后是想通过薛定谔方程正确的引导物理态的变化，那么现在这个哈密顿量定义是否合理呢"

这个问题比看起来深，要在 三个层次 上分别审视——物理上是否描述对了实验、数学上是否是良定义的算子、数值上是否适合直接喂给积分器。三件事都合格才算"合理"。

H(t) = ∑a[−½∂xa² + ½ω²xa²] + g∑a<bδ(xa−xb) + κ∑acos(2kLxa)·∑nδ(t−nT)

A2.2 数学层面 —— 合理，但要正确"解读"δ(t)

天真地看，把 H(t) = H0 + V·δ(t) 塞进薛定谔方程：

i∂t|ψ⟩ = [H0 + V·δ(t)]|ψ⟩

这在数学上 并不是 一个标准微分方程——因为 δ(t)|ψ⟩ 不是一个普通函数。正确的解读是 把它理解成算子值分布，通过对 δ 附近积分得到 jump condition：

|ψ(nT+)⟩ = exp(−iV)|ψ(nT−)⟩ = exp[−iκ·cos(2kLx)]|ψ(nT−)⟩

在 t ∈ (nT, (n+1)T) 之间，δ(t) 不激活，薛定谔方程退化为：

i∂t|ψ⟩ = H0|ψ⟩,     H0 = T + Vho + g·δ(xa−xb)

这是 普通 的连续时间演化。所以整个 Floquet 周期的严格数学定义是一个 算子乘积：

UFloquet = exp(−iH0T) · exp[−iκ·cos(2kLx)]
关键洞察。 写成 H = H0 + V·δ(t) 只是 Floquet 算子乘积的速记——真正严格的数学对象是那个乘积。把 δ(t) 看成"连续哈密顿量"是 误用，这正是 RK4 数值积分失败的深层原因：它在把一个 不连续演化 当 连续演化 来积分。

A2.3 数值层面 —— "合理"取决于你用什么算法

同一个物理哈密顿量可以有几种等价的数学表述，数值上表现差异巨大：

表述公式数值友好度备注
F1 连续 + δ(t) i∂tψ = [H0 + V·δ(t)]ψ ❌ 差 RK4 认为 δ(t) 是脉冲幅值发散，崩
F2 有限脉冲极限 i∂tψ = [H0 + (κ/Tp)cos·χ]ψ ⚠ 差 Tp→0 时刚性爆炸（Q1 已析）
F3 Floquet 算子乘积 ψn+1 = e−iH0T·e−iκcos·ψn ✅ 好 分成自由演化 + 解析 kick，各自最优算法
F4 相互作用绘景 i∂tψI = VI(t)·δ(t)·ψI ✅ 好 把 H0 吸收到绘景里，只剩 kick
F3 和 F4 数学上跟 F1 完全等价，只是哈密顿量的"组织方式"不同。物理一样，演化一样，能量谱一样，只是 代码实现的抓手不同。你现在代码里 run_kicked_gamma0_test 走的就是 F3：free_evolve_fixed_basis 精确做 e−iH0T，apply_analytic_kick 精确做 e−iκcos。失败的 run_kick_convergence_test 走的是 F2，所以糟。

A2.4 TDVP 对哈密顿量的"解读"又是另一层

哈密顿量定义对不对是一个问题，TDVP 能不能正确跟随这个哈密顿量 是另一个问题。TDVP 不是解薛定谔方程，是解 投影后的薛定谔方程：

i∂t|ψ(z)⟩ = PTψM · H|ψ(z)⟩

其中 PTψM 是到变分流形 M 在 ψ 处切空间的投影。于是 TDVP 给出的动力学 ≠ 真薛定谔动力学，差距是 (1−P)H|ψ⟩——被流形"看不见"的那部分。

  • 对 基态搜索（虚时 TDVP）：这没关系，只要流形含真基态附近就行。
  • 对 自由演化：只要 H0 保持态在流形上（比如谐振子基态附近），误差可控。
  • 对 kick 演化：kick 把态强力推出流形，(1−P)Ukickψ 巨大，TDVP 公式 数学上仍然成立（它就是产生最佳切空间近似），但 物理上是错的（因为近似质量差）。
所以"哈密顿量是否合理"和"TDVP 输出是否合理"是两件事。 哈密顿量本身（F1-F4 任一表述）都物理正确；但 TDVP 能把薛定谔演化 投 到变分流形上得到正确结果，依赖流形是否对 H 闭合。Part 2 讲过，ECG 流形对 kick 不闭合——这是 TDVP 失败的根因，不是哈密顿量本身的问题。

A2.5 改哈密顿量能解决问题吗？

理论上有几条路，但都不推荐。列一下以备参考：

重定义 H 的方式能否救 TDVP物理代价
把脉冲形状改光滑（高斯、sech²） ❌ 不能 无，Floquet 动力学首阶不变
把 cos 换成 polynomial approximation ⚠ 可能 改变物理，破坏动量周期性
做 rotating wave approximation ❌ 不适用 破坏 QKR 本质
换相互作用绘景 (F4) 部分可 无，只是换参考系，但 ECG 基底也随之变化，还要处理

结论是 不应该改哈密顿量——它已经是物理上正确的模型。应该改的是 算法如何利用这个哈密顿量的结构（Strategy A 就是利用"kick 可解析作用"这个结构）。

A2.6 用一句话回答

哈密顿量定义 100% 合理——它严格对应 Guo 等人实验的 QKR 模型，每一项都有明确实验对应。

但"哈密顿量合理"不等于"薛定谔演化可数值实现"。薛定谔方程 i∂tψ = Hψ 对 F1 表述是算子值分布方程，需要通过 Floquet 算子乘积（F3） 来正确解读，而不是当成普通 ODE 喂给 RK4。

"TDVP 能否正确引导态演化"又是一个独立问题——取决于变分流形对哈密顿量的闭合性，这跟哈密顿量定义无关，是流形 vs 算子的几何问题。

所以你现在的失败：哈密顿量没错，薛定谔方程没错，错的是把 Floquet 乘积结构（F3）当成连续 ODE（F2）来积分，并且用了一个对 kick 不闭合的变分流形。改 算法表述（走 F3 + Strategy A），不改哈密顿量。
Q3 · 空间 δ(x) 不违和，但时间 δ(t) 让 H 在那一点无穷大？
原问题。"空间的 delta 我不会引起这个问题是因为如果粒子确实演化到了有 delta 的位置，那么他的能量确实是那个数，但是时间的我很难理解，因为他不像空间，粒子步走到那个位置就是了，时间是一定会到那个时间，有这个时间的 delta 存在难道在那个点你的哈密顿量无穷大吗？"

你这个疑问是 量子力学里最深的一个不对称——时间和空间在 QM 里的地位根本不同。很多物理系学生卡过这点。我一层层解开。

A3.1 你的直觉抓对了一个关键不对称

确实有一个基本差异，你总结得很精准：

空间 δ(x − x₀)时间 δ(t − t₀)
"事件"是否一定发生 概率性——粒子只有在 x=x₀ 附近时才有效 决定性——时间一定会流过 t=t₀
出现在哈密顿量的位置 作为 势能项（哈密顿算子的一部分） 作为 时间系数（乘在算子前面）
直觉上是否"会让 H 无穷大" 只在点 x₀ 处，但粒子不一定在那 在 t₀ 一定发生，似乎 H→∞

答案是：是的，形式上 H(t) 在 t=t₀ 确实"无穷大"。但 这并不引起任何物理问题，原因需要从三个层面解释。

A3.2 空间 δ(x)：为什么不违和

考虑 1D 接触相互作用 Hint = g·δ(x1−x2)。你会问：在 x1=x2 处，这不是无穷大吗？

形式上是。但 能量期望 是：

⟨Hint⟩ = ∫dx1dx2 |ψ(x1,x2)|² · g·δ(x1−x2) = g·∫dx |ψ(x,x)|²

δ 被 概率密度 |ψ|² 积分掉了，变成有限数。物理上：δ 的无穷大权重 × 波函数在那一点的有限概率 = 有限能量。

这就是你说的"粒子确实演化到了有 δ 的位置，那么他的能量确实是那个数"——更准确说是"有 g·|ψ(x,x)|² 的能量密度贡献"。

A3.3 时间 δ(t)：形式上 H 确实无穷大

现在看时间 δ。哈密顿量 H(t) = H0 + V·δ(t−t0)。

在 t = t0：是的，H(t0) = ∞（作为分布）。但问题是——"H 在某一时刻的值"根本不是一个物理观测量。

这是关键澄清。 在 QM 里我们从不测"H 在某时刻的值"；我们测 H 的期望 ⟨ψ(t)|H|ψ(t)⟩，这个量对应"如果此刻把系统扔进能量测量装置，期望得到什么"。对于含 δ(t) 的 H，在 t=t0 那一瞬间 ⟨H⟩ 确实是发散的——但这个"瞬间"测不到（δ 函数宽度为零），所以不是物理量。

物理量应该是：

  • "kick 之前" 的能量: E− = ⟨ψ(t0−)|H0|ψ(t0−)⟩
  • "kick 之后" 的能量: E+ = ⟨ψ(t0+)|H0|ψ(t0+)⟩
  • "注入的能量": ΔE = E+ − E− ← 有限！

你代码里 compute_total_energy(basis, terms_free) 算的就是 ⟨H0⟩，从来不把 kicking 项算进"能量"——这很对，因为 kick 本身不是能量，只是把能量注入进来的 瞬时过程。

A3.4 为什么形式上的无穷大不引起物理问题

薛定谔方程：

i·∂t|ψ⟩ = H(t)|ψ⟩

在 t0 附近的小邻域 (t0−ε, t0+ε) 内积分：

i·[|ψ(t0+ε)⟩ − |ψ(t0−ε)⟩] = ∫t0−εt0+ε H(t)|ψ⟩ dt
= ∫ H0|ψ⟩ dt + V·|ψ(t0)⟩ · ∫δ(t−t0)dt

当 ε→0：左边是 |ψ⟩ 的跳变；右边第一项 → 0（H0 有限，对零宽度积分为零）；右边第二项 = V·|ψ(t0)⟩。所以：

|ψ(t0+)⟩ − |ψ(t0−)⟩ = −i·V·|ψ(t0)⟩

但这个线性关系不是幺正的。真正的严格结果是（对指数化后的幺正形式做积分）：

|ψ(t0+)⟩ = exp(−iV)·|ψ(t0−)⟩

这是 有限的、幺正的、norm 守恒的 跳变。V 可以取任意有限算子，δ(t) 的无穷大 没有出现在可观测量里——它被指数化"吃掉"了。

A3.5 经典力学的完美类比

想象一个经典粒子受到 冲量力 F(t) = p0·δ(t)。

  • 在 t=0：力无穷大（形式上）
  • 但牛顿第二定律 dp/dt = F 积分：Δp = ∫F dt = p0
  • 动量跳变 有限，能量跳变 有限，位置 不变（因为 ∫v dt → 0）
核心洞察。 在经典力学里我们早就接受了"冲量力"：锤子敲球的一瞬间，力公式上无穷大，但动量改变有限。没人觉得"锤子那一瞬间力无穷大"是物理问题——因为 "瞬时的力值"不是物理观测量，"动量改变"才是。

量子 δ(t) kick 完全类比："瞬时的 H 值"不是物理观测量，"状态的幺正跳变 exp(−iV)"才是。

A3.6 空间 vs 时间 δ 的数学对比

空间 δ(x)时间 δ(t)
数学对象 势能算子里的分布 哈密顿量的时间权重分布
如何被"积掉" 对 |ψ(x)|² 积分（空间期望） 对 dt 积分（时间演化生成）
积出来的有限量 能量期望 g·|ψ(x0)|² 幺正跳变 exp(−iV)
物理观测的是 稳态能量（标量） 状态跳变（算子）
"那一点 H 无穷大吗" 形式上是，但被 |ψ|² 权重化 形式上是，但不是可观测量

所以你观察到的不对称 确实存在，但它反映的是 时间在 QM 里是参数而不是算子（Pauli 定理：不存在"时间算子"）。时间不是可观测量，所以 "H(t) 在某时刻的值" 也不是可观测量。

A3.7 再更深一层：时间作为"Floquet 操作"

对周期性 δ-kick，最清晰的物理图像不是"H(t) 在不同时刻取不同值"，而是：

每周期 T 执行一次算法： (1) 用 H0 自由演化时间 T (2) 把状态乘以 exp(−iκ·cos(2kLx)) 循环……

这里根本 没有 δ(t)——它只是记法。实际演化是 "连续演化 + 离散跳变" 的 混合系统（hybrid system）。时间被分为两种：连续流动时间（由 H0 支配）和离散事件时间（由 kick 支配）。

A3.8 一句话回答

"H 在那一点无穷大吗？"——形式上是，但这不是物理观测量，所以不引起任何问题。

空间 δ(x) 被 概率密度 |ψ|² 积成有限能量；时间 δ(t) 被 时间 dt 积成有限幺正跳变 exp(−iV)。两者的不对称在于 积分对象不同：空间 δ 积掉后得到能量（数），时间 δ 积掉后得到算子（演化）。

你的直觉——"时间一定会到"——抓对了一个关键点：时间 δ 的事件是 确定性 的，每个周期必然发生一次。但 "事件发生" 和 "H 在那一点无穷大" 是两回事。事件发生是 exp(−iV) 这个有限幺正算子被施加；形式上的"无穷大"只是这个幺正算子的"生成元"（generator）在零时长里的极限写法。

量子 δ(t) ≈ 经典冲量力 F·δ(t)：锤子敲球一瞬间"力无穷大"，但动量变化有限。kick 一瞬间"H 无穷大"，但状态跳变有限。这是同一个数学结构，只是从经典观测量（动量）换成了量子观测量（幺正）。

A3.9 与我们代码失败的联系

回到 ECG-TDVP 失败：

  • 物理上 δ(t) 完全合法——它定义了一个有限幺正跳变，状态变化有限，能量变化有限
  • 数值上问题不是"H 无穷大"——而是我们 假装 H(t) 是连续函数（用 κ/Tpulse 来光滑化），然后 RK4 被这个"光滑化后的大数"噎死
  • 正确做法是直接计算幺正跳变 exp(−iκcos(2kLx))，绕开 H 的瞬时值（这正是 Strategy A 做的事）

所以你的疑问其实帮我们再次定位了问题所在：不要去"积分"含 δ(t) 的 H，要直接"施加"它生成的幺正跳变。H 在那一点是否无穷大是个伪问题；exp(−iV) 作为一个算子永远是有限的幺正算子，这才是物理。

Q4 · 固定 K，用 TDVP 演化 R 去适应 kick 的不断注入？
原问题。"如何在固定基底数量的情况下，用 TDVP 演化 R 的值去适应我们 kick 的不断注入"

这是个非常关键的物理-算法问题。先把问题数学上说清楚，再提两条具体路线。

A4.1 问题的数学表述

我们要解的是：

给定真实被踢态 |ψkicked⟩ = Ukick|ψ0⟩（它"应该"由 R-移位的 ECG 组成），求一组 固定 K 个 ECG 参数 {ui, Ai, Bi, Ri}i=1..K，使得 |ψECG(z)⟩ 最佳逼近 |ψkicked⟩。

注意这 不是标准 TDVP 问题。标准 TDVP 是"求 ż 让态沿薛定谔方程演化"。我们这里没有时间，是个 纯变分拟合 问题。

但是有一个重要的转化：可以把"拟合到目标态"伪装成 一个 TDVP 问题。

A4.2 关键数学：把"拟合"变成"虚时间 TDVP"

定义一个 人造哈密顿量：

Heff = −|ψkicked⟩⟨ψkicked|

这是个投影算符（带负号）。它的最低本征态就是 |ψkicked⟩ 本身（本征值 −1）。所以 对 Heff 跑虚时间 TDVP，参数会流向 |ψkicked⟩ 的最佳变分逼近。

虚时间方程：

żα = −Cαβ−1 · gβ,     gα = ∂⟨Heff⟩ / ∂zα*

其中：

⟨Heff⟩ = −|⟨ψ(z)|ψkicked⟩|² / ⟨ψ(z)|ψ(z)⟩     (负的 fidelity 平方)

梯度 g 是：

gα = − ⟨∂αψ(z)|ψkicked⟩ · ⟨ψkicked|ψ(z)⟩ / ⟨ψ|ψ⟩ + (归一化项)

度量 C 跟标准 TDVP 完全一样（只取决于 ansatz 几何，不取决于 H）。

A4.3 关键事实：|ψkicked⟩ 是封闭可算的

虽然它"应该长在新的 R 上"，但因为 ECG 流形对乘动量平面波闭合，我们有 精确的解析表达：

|ψkicked⟩ = ∑j=1..K uj(0) · ∑n=−nmax+nmax (−i)n Jn(κ) · |φRj + i·n·kL/Bj⟩

总共 K × (2nmax+1) 个 ECG 项，每项的参数都是已知的。所以：

  • ⟨ψkicked|ψkicked⟩ 是高斯积分（数值常量）
  • ⟨φi(z)|ψkicked⟩ 也是高斯积分（取决于 z）
  • 梯度 ∂α⟨ψ(z)|ψkicked⟩ 也都是高斯积分的导数
所有量在 ECG 框架下都是闭式的，不需要任何数值积分。这一点是 ECG 基底相对于其他基底的核心优势。

A4.4 两条具体实现路线

路线 1 — "Target-fitting TDVP"（最纯净）

按上面的数学路线，新写一个 target_fitting_evolution：

  • 输入：当前 basis、目标 |ψkicked⟩（用 K·(2nmax+1) 个 ECG 表达）
  • 步骤：
    1. 算 C（完全复用现有 assemble_C）
    2. 算 g（新代码：跨基底 ECG-ECG overlap 的导数）
    3. 解 C·ż = −g，更新 z
    4. 重复直到 fidelity 收敛
优点缺点
  • 严格的变分最优解
  • 不会有 RK4 刚性问题（虚时间梯度下降）
  • 直接告诉你"K 个 ECG 能做到的最佳 fidelity 是多少"
  • 需要写跨基底 overlap 梯度（≈100 行新代码）
  • 全新概念，跟现有 TDVP 流程不能直接复用

路线 2 — "有限脉冲 + 实时 TDVP"（复用现有代码）

把瞬时 kick 换成有限宽度 Tpulse 的脉冲：

  • 脉冲期间 H = T + Vho + (κ/Tpulse)·cos(2kLx)
  • 用 realtime_tdvp_step 跑时间 Tpulse，让 R, A, B, u 都参与（optimize_R=true 等）
  • 选 不太小 的 Tpulse（比如 0.1）避免刚性

这条路线的核心不是"直接找最像 kicked 态的固定-K ECG"，而是先把理想瞬时 kick 正则化 成一个短时真实脉冲，然后让 TDVP 在固定基底流形上去跟踪这段短时薛定谔演化。

(i) 从理想 δ-kick 到有限脉冲

理想 kick 的传播子是：

Ukick = exp[−iκV],     V = ∑a cos(2kLxa)

路线 2 用一个面积固定、宽度为 Tpulse 的矩形脉冲去逼近它：

Hpulse = H0 + (κ/Tpulse)V,     0 < t < Tpulse

其中 H0 = T + Vho (+Vint) 是非 kick 的那部分。单个脉冲的精确传播子是

Upulse(Tpulse) = exp[−i(H0 + κV/Tpulse)Tpulse] = exp[−i(κV + TpulseH0)]

所以当 Tpulse → 0 时，Upulse → exp(−iκV) = Ukick。这就是路线 2 的物理基础：把瞬时跳变改写成一个零宽度极限下的真实短脉冲。

如果实现上继续沿用当前 run_kick_convergence_test 的"kick-only pulse" 简化，那就相当于把上式里的 H0 暂时设成 0；推导结构不变，但那更接近数值正则化，而不是严格的有限脉冲物理。

注意 H0 与 V 一般不对易，所以有限脉冲相对理想瞬时 kick 的修正可由 BCH/Magnus 展开写成：

Upulse = e−iκV exp[−iTpulseH0 + (κTpulse/2)[V,H0] + O(Tpulse2)]

因此对当前这种非对称矩形 pulse，相对理想 δ-kick 的系统偏差一般是 O(Tpulse)；只有把 pulse 做成对称中心化，或把整个单周期写成 Strang 结构时，主误差才会抬到 O(Tpulse2)。

(ii) TDVP 在 pulse 窗口里的投影方程

设固定-K ECG 流形上的态写成 |ψ(z)⟩，其中 z = (u, A, B, R)。实时 TDVP 在切空间里寻找一个速度

|ψ̇⟩ = ∑β żβ |∂βψ⟩

使得残差 ‖\,i|ψ̇⟩ − H|ψ⟩‖ 最小。McLachlan / Dirac-Frenkel 变分原理给出：

∑β Cαβ(z) żβ = −i gα(z)
Cαβ(z) = ⟨∂αψ|P⊥|∂βψ⟩,     gα(z) = ⟨∂αψ|H|ψ⟩

把 pulse 哈密顿量代进去：

gα(z) = gα(0)(z) + (κ/Tpulse) gα(V)(z)
gα(0) = ⟨∂αψ|H0|ψ⟩,     gα(V) = ⟨∂αψ|V|ψ⟩

于是参数动力学是

C(z)ż = −i g(0)(z) − i(κ/Tpulse) g(V)(z)

这条式子直接解释了 pulse-window 的数值刚性：g(V) 被 1/Tpulse 放大，而 C 只由流形几何决定，不随 Tpulse 缩放。

(iii) 为什么 1/Tpulse 很大，但总 kick 量并不发散

把时间重标度成 τ = t/Tpulse ∈ [0,1]，则 ż = (1/Tpulse) dz/dτ。上式变成：

C(z) dz/dτ = −i[Tpulseg(0)(z) + κg(V)(z)]

取 Tpulse → 0，得到有限极限：

C(z) dz/dτ = −iκ g(V)(z),     τ ∈ [0,1]

所以路线 2 的真正极限不是"参数速度无穷大"，而是一条有限冲量映射：瞬时速度的确是 O(1/Tpulse)，但总位移 Δz = ∫żdt 仍是 O(1)。

若再对 κ 做小参数展开，在起点 z− 附近有：

z+ = z− − iκ C(z−)−1 g(V)(z−) + O(κ2) + O(Tpulse)

这就是固定流形上"一次 kick 把参数推到哪里去"的一阶公式。

(iv) 在 ECG 里，这条路线实际上在推动什么自由度

ECG 有一个关键闭合性质：平面波因子会把中心参数 R 往虚方向平移。对单粒子分量，

eiqxa φR ∝ φR + (iq/2)B−1ea

取 q = 2nkL 后就是

ei2nkLxa φR ∝ φR + inkLB−1ea

而理想 kick 的 Jacobi-Anger 展开是

e−iκ cos(2kLxa) = ∑n=−∞∞ (−i)n Jn(κ) ei2nkLxa

因此真实被踢态会在很多不同的虚 R 扇区上展开。路线 2 的 TDVP 所做的事，是让现有这 K 个基函数的 u, A, B, R 连续移动，去尽量跟随这个多扇区展开；但它并不会像 Strategy C 那样真的把每个 Bessel 扇区都显式生出来。

这也解释了为什么 optimize_R=true 对这条路线尤其关键：kick 最先想推动的，正是 R 的虚部，也就是动量自由度。

(v) 为什么路线 2 不等于路线 1 的最优拟合

路线 1 是一个全局变分拟合：直接在整个固定-K 流形里找 fidelity 最大的点。路线 2 则是一个局部投影动力学：在每个时刻只保留哈密顿量作用在切空间里的那部分。

i|ψ̇TDVP⟩ = PTψM H |ψ⟩

被漏掉的部分是

|r⟩ = (1 − PTψM)(−iH|ψ⟩)

在 pulse 窗口里 H = H0 + (κ/Tpulse)V，所以整个 pulse 的累计投影误差量级是

∫0Tpulse ‖r(t)‖dt ≈ κ ‖(1−P)V|ψ⟩‖ + O(Tpulse)

关键点在这里：把 Tpulse 取得更小，并不会自动消掉流形不匹配误差，因为 1/Tpulse 的放大和 Tpulse 的积分宽度会互相抵消。要让路线 2 真正接近理想 kick，根本上仍然要靠流形更会表示 V|ψ⟩：例如增大 K、开放 R 的虚部、或直接显式引入动量增广。

一句话概括路线 2。 它不是"一步投到最优 kicked 态"，而是"用有限脉冲把理想 kick 正则化，然后在固定-K ECG 流形上解投影后的短时实时薛定谔方程"。因此它保留了完整 TDVP 的物理语义，但 fidelity 的上限仍由流形表达能力决定。
优点缺点
  • 完全复用现有 TDVP 代码
  • 物理上是"真薛定谔演化"（没有人造哈密顿量）
  • 自然把 kick 拆成多步，每步都是标准 TDVP
  • 仍需小心 Tpulse 选择（太小会 RK4 不稳）
  • 当前这种非对称矩形 pulse 相对理想 δ-kick 的系统偏差一般是 O(Tpulse)；若改成对称 pulse / Strang 结构，可提升到 O(Tpulse2)
  • 跟 Trial 1 失败的路径很像，需要确认这次能稳

A4.5 我的推荐 — 先走路线 1

理由：

  1. 数学上严格：直接定义"K 个 ECG 对被踢态的最佳逼近"，没有歧义
  2. 可对照路线 2：路线 1 的解就是路线 2 的"理论上界"——后者无论怎么调 Tpulse，最多就达到这个 fidelity
  3. 作为 Strategy C 的便宜替代：不需要真正扩张基底，但效果接近
路线 2 之后做对比验证，能告诉我们"实时 TDVP 在不增加基底情况下能逼近变分最优解到多少"。

A4.6 路线 1 详细数学推导

这一节把路线 1 (Target-fitting TDVP) 的数学一步一步走完，让你能看清每个量在算什么、什么是新代码、什么可以复用。

A4.6.0 先回答："为什么还在说虚时间？这不就是优化吗？"

你的直觉对了。 我们做的本质是 变分优化（在 K 个 ECG 流形上找 fidelity 最大的参数），不是"量子时间演化"。"虚时间 + 虚拟哈密顿量"这套话术只是个 借口——它和"对负 fidelity 做自然梯度下降"是 完全一样 的算法，只是命名不同。

同一件事的两套语言：

说法 1: 自然梯度下降（优化语言）说法 2: 虚时间 TDVP（量子语言）
核心对象 损失函数 L(z) = −F(z) 虚拟哈密顿量 Heff = −|ψT⟩⟨ψT|
动机 "找 z 让 L 最小" "找 Heff 的基态（变分逼近）"
更新公式 z ← z − η · C−1 · ∇L z ← z + dτ · ż,   C·ż = −g
步长叫法 学习率 η 虚时间步长 dτ
度量 C Fisher / Fubini-Study 自然度量 切空间 Gram 矩阵
梯度 g 损失对 z* 的导数 ∇L 能量对 z* 的导数
收敛准则 |∇L| → 0 或 |ΔL| < tol ⟨H⟩ 平台

把它们放在一起看就是同一个数学对象——只是历史上量子物理界用 H 和"虚时间"语言，机器学习界用 L 和"梯度下降"语言。

那为什么我用 TDVP 语言？ 三条工程理由：

  1. 能直接复用现有 C++ 代码：assemble_C、tdvp_step、evolution 主循环、Tikhonov 正则化、SVD 求解、自适应步长……所有这些都是按 TDVP 框架写的，输入 H 输出 ż。把 H 换成 Heff 后整套机器原封不动。
  2. SolverConfig 已经全配好：lambda_C、rcond、dtao_grow、能量收敛判据……这些都是 TDVP 的"参数旋钮"，跟优化里的"hyperparameters"对应得很好。
  3. 文献兼容：ECG / 高斯波包变分文献基本都用 TDVP 语言，方便交流。
所以——如果你心里把"虚时间 TDVP"翻译成"用 C 矩阵作度量的自然梯度下降"，理解是 100% 正确的。下面的推导我继续用 TDVP 语言（C·ż = −g 这种），但你脑子里随时换成 ż = −η·C−1·∇L 也一样。

A4.6.1 变量与目标态：每个量到底是什么数

把抽象记号落到具体数字上，按四步走：

(a) 变分参数 z 是什么？

K 个 ECG 基函数（N=1 单粒子情形），每个基函数有 4 个复参数：

参数含义对 N=1 是…
ui线性叠加系数1 个复数
Ai高斯指数里的"平方项"矩阵元1×1 矩阵 = 1 个复数
Bi另一个平方项矩阵元（与 A 联动，指数里以 A+B 形式出现）1×1 矩阵 = 1 个复数
Ri中心位置（实部 = 空间位置, 虚部 = 动量相位）N 维向量 = 1 个复数

K 个基函数共 4K 个复参数 = 8K 实参数。这一整组数就是 z。

具体例子（K=1, 任务 A 跑完后的最优值）：

z = (u1, A1, B1, R1) ≈ (1.000, 0.500, 0.000, 0.000)    [全实]

对应物理：φ(x) = exp[−(A+B)x²+2RBx−R²B] = exp(−0.5 x²)，正是谐振子基态（再乘 u₁=1 总系数）。

(b) 当前 ECG 态 |ψ(z)⟩ 是什么？

|ψ(z)⟩ = ∑i=1..K ui · |φ(Ai, Bi, Ri)⟩

K=1 时就是单个加权高斯：

ψ(x; z) = u1 · exp[−(A1+B1)x² + 2R1B1x − R1²B1]

K>1 时是几个高斯的线性叠加。z 是优化变量——优化过程中所有这些数都会变。

(c) 目标态 |ψT⟩ 是怎么造出来的？

物理上：|ψT⟩ = Ukick·|ψ0⟩——基态被踢一次的"真实"结果。

第 1 步: Jacobi-Anger 把 Ukick 展开：

e−iκ cos(2kLx) = ∑n=−∞..+∞ (−i)n Jn(κ) · ei·2nkLx

每一项是一个动量平面波 ei·2nkLx 乘 Bessel 系数。

第 2 步: 每个动量平面波乘到一个 ECG 上, 等于 R 虚部移位:

ei·q·x · φ(A, B, R) = (常数因子) · φ(A, B, R + i·q/(2B))

—— 这就是策略 HTML 里 Part 2 的核心事实："ECG 流形对乘动量平面波闭合"。

第 3 步: 全部组装起来。 假设初始基态是 K=1 单 ECG，参数 (A₁=0, B₁=0.5, R₁=0, u₁(0)=1)，这样 A+B=0.5 给出 exp(−x²/2)；κ=1, kL=0.5：

动量移位量 q = 2nkL = n    (因 kL=0.5)
R 移位量 = i·q/(2B) = i·n/(2·0.5) = i·n

所以截断到 |n|≤4 的目标态有 9 个 ECG 项（n = −4, …, +4）：

n系数 cn = (−i)n·Jn(κ=1)(A, B, R) 组合
0+0.7652(0, 0.5, 0)
+1−0.4401i(0, 0.5, +i)
−1−0.4401i(0, 0.5, −i)
+2−0.1149(0, 0.5, +2i)
−2−0.1149(0, 0.5, −2i)
+3+0.0196i(0, 0.5, +3i)
−3+0.0196i(0, 0.5, −3i)
+4+0.0025(0, 0.5, +4i)
−4+0.0025(0, 0.5, −4i)

简记成统一公式：

|ψT⟩ = ∑m=1..M cm · |χm⟩,    M = 9

其中 cm 和 |χm⟩（即 (A(T)m, B(T)m, R(T)m)） 全部已知，全部固定，整个优化过程中不动。

对一般 K： 如果初始基有 K 个 ECG（不止 1 个），那目标态有 K·(2nmax+1) 个 ECG 项。M = 9K 之类。每一项都来自某个原基函数的 R 移位。

(d) 优化要做什么？

当前态 |ψ(z)⟩目标态 |ψT⟩
基函数数量K （我们手头容量）M = K(2nmax+1) （物理要求）
R 的取值z 中的待优化参数固定在 0, ±i, ±2i, …, ±nmaxi
系数ui 待优化cm 固定 (Bessel 加权)
状态能调整要逼近
核心矛盾。 目标态有 M=9 个动量扇区 (R = 0, ±i, ±2i, ±3i, ±4i)。我们手头只有 K 个 ECG 容量。
  • K=1: 一个 ECG 没法同时占据 9 个扇区。最佳策略是把 R 落在 0 附近、并把 A+B 调整成"虚拟覆盖"多扇区的宽度。fidelity 一定 < 1。
  • K=3: 大致可以让 3 个 ECG 各占一个主要动量扇区（比如 R≈0, R≈+i, R≈−i），fidelity 显著提升。
  • K=9: 理论上可以让每个 ECG 各占一个 R 扇区，完美匹配，fidelity → 1。
这就是 R 在 TDVP 中应该被允许移动的物理原因——固定 R=0 的话, 单 ECG 永远没法表达多动量扇区。

所以路线 1 干的事就是：让 z 中的 R 自动移到这些"最佳协调位置"，同时也调 A、B、u 让总叠加最像 ψT。

A4.6.2 目标函数：保真度 → 虚拟哈密顿量

我们要 最大化 保真度（K 个 ECG 表达 |ψT⟩ 的好坏）：

F(z) = |⟨ψ(z)|ψT⟩|² / [⟨ψ(z)|ψ(z)⟩ · ⟨ψT|ψT⟩]

分母里 ⟨ψT|ψT⟩ 是常量，可以扔掉。等价地最大化：

F̃(z) = |⟨ψ(z)|ψT⟩|² / ⟨ψ(z)|ψ(z)⟩

关键变换：定义虚拟哈密顿量 Heff = −|ψT⟩⟨ψT|。则：

⟨Heff⟩ ≡ ⟨ψ|Heff|ψ⟩/⟨ψ|ψ⟩ = −⟨ψ|ψT⟩⟨ψT|ψ⟩/⟨ψ|ψ⟩ = −F̃(z)

所以 "最小化 ⟨Heff⟩" ⇔ "最大化 F̃"。这正是标准的"求基态"问题——只不过 Heff 是个特殊的投影算符。它的本征值结构是：

本征态本征值
|ψT⟩/‖ψT‖（归一化目标态）−1
所有跟 |ψT⟩ 正交的态0

非常干净：基态唯一，能量 −1。任何对它跑虚时间演化的算法都会流向这个基态。

A4.6.3 虚时间 TDVP 的方程形式

标准虚时间薛定谔方程（无投影版）：

∂τ |ψ⟩ = −H |ψ⟩

把状态限制在变分流形上，TDVP 投影到切空间（这步跟已有代码完全一样）：

PTψM · ∂τ|ψ(z)⟩ = −PTψM · H|ψ(z)⟩

用参数 z 写，得到 TDVP 方程组：

∑β Cαβ · żβ = −gα

其中：

  • 度量 Cαβ：跟标准 TDVP 完全一样，只取决于 ansatz 的几何（高斯参数），跟 H 无关。直接复用现有 assemble_C(alpha_z_list, basis)。
  • 梯度 gα：依赖于 H。这里 H = Heff，所以是 新代码。

具体地：

Cαβ = ⟨∂αψ | (1 − Pψ) | ∂βψ⟩     (复用)
gα = ∂⟨Heff⟩ / ∂zα*     (新写)

A4.6.4 梯度 gα 的链式法则展开

把目标函数拆成三个量来管：

P(z*) = ⟨ψ(z) | ψT⟩     (跟目标态的 overlap, z* 的函数)
P*(z) = ⟨ψT | ψ(z)⟩     (它的复共轭, z 的函数)
S(z, z*) = ⟨ψ(z) | ψ(z)⟩     (自我 overlap, 双线性)

则 ⟨Heff⟩ = −P · P* / S。对 zα* 取偏导（Wirtinger 微积分，把 z 和 z* 当成独立变量）：

gα = ∂(−P P*/S)/∂zα* = −(∂P/∂zα*) · P*/S + P P*/S² · (∂S/∂zα*)

把 P*/S 提出来更紧凑：

gα = −(P*/S) · [ ∂P/∂zα* − (P/S) · ∂S/∂zα* ]

方括号里的东西是 归一化校正后的"目标方向"——告诉你"为了让 |ψ⟩ 更像 |ψT⟩，zα 该往哪个方向走"。

A4.6.5 各部件的解析表达式

(i) P = ⟨ψ(z)|ψT⟩：跨基底 overlap 求和

P = ∑i=1..K ∑m=1..M ui* · cm · ⟨φi(z) | χm⟩

共 K·M = K²(2nmax+1) 个 ECG-ECG overlap，每个都是 闭式高斯积分。这就是新代码要算的核心。

(ii) ∂P/∂ui*：对 u 的导数最简单（u 出现在线性位置）

∂P/∂ui* = ∑m cm · ⟨φi(z) | χm⟩

就是 P 的 i 那一行 / ui*。

(iii) ∂P/∂Ai*, ∂P/∂Bi*, ∂P/∂Ri*：对 ECG 几何参数的导数

∂P/∂Ai* = ui* · ∑m cm · ∂⟨φi(z)|χm⟩/∂Ai*

同理 B、R。这些是新东西——现有代码的 derivative_overlap_partial 算的是同一组基内的 overlap 导数；这里需要的是 跨基（z 基对 χ 基）的 overlap 导数。但数学结构相同，公式可以照搬高斯积分的 Wirtinger 求导规则。

(iv) S = ⟨ψ|ψ⟩ 和 ∂S/∂zα*：完全复用现有代码

这是标准 TDVP 用的"自身 overlap"和它的导数，overlap() 和已有的导数函数都能直接给。

(v) C 矩阵：完全复用 assemble_C(alpha_z_list, basis)。

A4.6.6 算法伪代码

target_fitting_evolution(basis_init, target_components, max_steps, dτ):

    z = basis_init                              # 当前 ECG 参数（K 个基函数）
    targets = target_components                 # M 个固定的 ECG (c_m, |χ_m⟩)

    for step in 1 .. max_steps:

        # 1. 度量（复用现有代码）
        C = assemble_C(z)                       # (K·d) × (K·d) Hermitian

        # 2. 跟目标的 overlap（新代码: 跨基底 ECG-ECG）
        P  = sum_{i,m} u_i^* · c_m · ⟨φ_i(z)|χ_m⟩
        S  = ⟨ψ(z)|ψ(z)⟩                       # 自身 overlap, 复用

        # 3. 梯度（新代码: 跨基 overlap 的 z* 导数）
        dP_dzs = grad_overlap_target(z, targets)    # (K·d,) 复向量
        dS_dzs = grad_overlap_self(z)               # (K·d,) 复向量, 复用

        g = -conj(P)/S · (dP_dzs - (P/S) · dS_dzs)

        # 4. 解 TDVP 方程 (Tikhonov 正则化 + SVD, 跟现有 evolution 一样)
        dz = solve(C · dz = -g, lambda_C)

        # 5. 更新参数（虚时间步）
        z = z + dτ · dz

        # 6. 收敛判据
        F = |P|² / (S · ||target||²)
        if 1 - F < tol: break

    return z, F_final

A4.6.8 几个工程细节

  1. 正则化：C 在某些参数方向上可能病态（基函数共线、零模等）。Tikhonov 加 λI，或用 SVD 截掉小奇异值——这两件现有代码都已实现。
  2. 步长自适应：起始 dτ = 10⁻³，每成功一步 ×1.5；线搜索失败就减半。完全照搬 evolution() 里的逻辑。
  3. 归一化：每步后或周期性把 u 重新归一化让 ‖ψ(z)‖ = 1，避免数值漂移。
  4. 初值：从 SVM 基态出发，或者先做一次 apply_analytic_kick（路线 0：u 投影），把 z 推到一个"已经不算太差"的起点，再让 target-fitting 把 R/A/B 调到最优。
  5. 收敛判据：不再用 |dE| < tol（那是能量收敛），改用 |1−F| < tol（保真度饱和）。Final F 直接告诉你"K 够不够"。

A4.6.9 一句话物理图像

把"用 ECG 拟合被踢态"伪装成"求虚拟哈密顿量 Heff = −|ψT⟩⟨ψT| 的基态"，然后跑标准虚时间 TDVP——就借用了所有现成的 TDVP 机器（C 矩阵、SVD 求解、自适应步长），只换了梯度公式。新东西只有"跨基 ECG-ECG overlap 及其导数"，其他完全复用。

最终输出是 K 个 ECG 参数 {ui, Ai, Bi, Ri}，配合保真度 Fmax。Fmax = 1 表示 K 完全够用；Fmax < 1 量化告诉你"K 个 ECG 流形最多能表达 |ψkicked⟩ 到哪一步"——这就是 ECG 拟合的根本极限，跟具体算法无关。

附加说明 · 路线 1 里的 gα 到底怎么算

这里把路线 1（Target-fitting TDVP）里最关键、也最容易卡住实现的一步单独拎出来：gα 不需要像真实哈密顿量那样分别去算 kinetic / harmonic / delta / kick 的偏导；因为这里用的是一个 rank-1 投影型人工哈密顿量，所以梯度会塌缩成"一个 overlap 标量 × 一个 overlap 导数"。

G1. 目标态先固定成一个截断后的 ECG 和

设 kick 后的目标态已经用 Jacobi-Anger 展开并截断到 |n| ≤ nmax：

|ψtar⟩ = |ψkicked(nmax)⟩ = ∑m=1M cm|χm⟩,     M = K(2nmax+1)

这里每个 |χm⟩ 都是一个固定参数的 ECG，来自原始第 j 个基函数被某个 Bessel 扇区 n 踢出去后的分量；cm 是对应的已知复系数（含 (−i)nJn(κ) 和原本的 uj(0)）。

当前正在优化的 trial 态仍然是固定-K 的 ECG ansatz：

|ψ(z)⟩ = |ψ(u, A, B, R)⟩

关键点：|ψtar⟩ 是固定靶子，|ψ(z)⟩ 是要被 TDVP 推着走的变量态。

G2. 用目标态造一个投影型人工哈密顿量

最自然的定义是：

Heff = − |ψtar⟩⟨ψtar| / ⟨ψtar|ψtar⟩

记

Ntar = ⟨ψtar|ψtar⟩

则对任意 trial 态的能量泛函是

⟨Heff⟩ = ⟨ψ(z)|Heff|ψ(z)⟩ = − |⟨ψtar|ψ(z)⟩|² / Ntar

如果再除以 ⟨ψ(z)|ψ(z)⟩，它就是 fidelity 的负数；因此对这个 Heff 跑虚时间 TDVP，本质上就是在提升当前 ansatz 对目标态的 overlap / fidelity。

G3. gα 的闭式公式

路线 1 里最直接的定义是

gα = ⟨∂αψ(z)|Heff|ψ(z)⟩

把上面的 Heff 代进去：

gα = − ⟨∂αψ(z)|ψtar⟩ ⟨ψtar|ψ(z)⟩ / Ntar

定义两个中间量：

A(z) = ⟨ψtar|ψ(z)⟩
bα(z) = ⟨∂αψ(z)|ψtar⟩

于是整个梯度立刻化成

gα = − (A(z) / Ntar) · bα(z)

这就是实现上的核心简化：不再是"求真实哈密顿量的 partial"，而是"求目标态 overlap 的导数"。

如果你事先把目标态归一化成 Ntar = 1，那就更简单：

gα = − A(z) · bα(z)

G4. A(z) 和 bα(z) 怎么拆成 ECG-ECG 积分

因为目标态已经写成固定 ECG 的和：

|ψtar⟩ = ∑m cm|χm⟩

所以 overlap 标量是

A(z) = ⟨ψtar|ψ(z)⟩ = ∑m cm* ⟨χm|ψ(z)⟩

而导数项是

bα(z) = ⟨∂αψ(z)|ψtar⟩ = ∑m cm ⟨∂αψ(z)|χm⟩

因此实现上真正需要的新对象只有两类：

  • 普通 cross overlap：⟨χm|φi(z)⟩
  • cross overlap 的参数导数：⟨χm|∂αφi(z)⟩ 或等价的 bra 形式

这两类对象在 ECG 框架下依然都是高斯积分闭式，因此路线 1 的新代码重点不是新物理，而是跨基底 overlap / overlap-derivative kernel。

G5. 对不同参数 α，bα 实际代表什么

如果 α 取不同类型的变分参数，那么 bα 就对应不同的导数核：

  • α = ui：就是普通 overlap，本质上没有额外几何复杂度
  • α = Ri,a：是对第 i 个基函数的中心参数 Ri,a 求导后的 overlap
  • α = Ai,ab：是对宽度矩阵元 Ai,ab 求导后的 overlap
  • α = Bi,a：是对对角矩阵元 Bi,a 求导后的 overlap

所以 html 里之前提到的"需要写跨基底 overlap 梯度"，说的就是这些 bα，而不是重新写一整套 kinetic / harmonic / delta / kick 的哈密顿量导数。

G6. 代码层面的最小实现流程

  1. 先把 Jacobi-Anger 截断后的目标态存成 target_basis（每项一个固定 ECG + 一个复系数 c_m）。
  2. 计算目标态 norm：Ntar = ⟨ψtar|ψtar⟩。
  3. 对当前 trial basis 计算：A = ⟨ψtar|ψ(z)⟩。
  4. 对每个变分参数 α 计算：bα = ⟨∂αψ(z)|ψtar⟩。
  5. 组装：gα = −(A/Ntar) bα。
  6. 最后解虚时间 TDVP 线性系统：C(z) ż = −g(z)。

G7. 为什么有时会看到"归一化项"

如果你不是从 Heff 出发，而是直接对 fidelity

F(z) = |⟨ψtar|ψ(z)⟩|² / ⟨ψ(z)|ψ(z)⟩

求导，就会多出

∂α*⟨ψ(z)|ψ(z)⟩

这样的归一化项。把它完整写开是：

gα = − (A*/S) [ ∂α*A − (A/S) ∂α*S ],     S = ⟨ψ(z)|ψ(z)⟩

但如果你使用的是 TDVP 里本来就在用的投影度量

Cαβ = ⟨∂αψ|P⊥|∂βψ⟩,     P⊥ = 1 − |ψ⟩⟨ψ| / ⟨ψ|ψ⟩

这个归一化方向本来就会被投掉。因此从实现角度看，最干净的仍然是前面的 rank-1 写法：

gα = − (A/Ntar) bα
一句话总结。 路线 1 的 gα 本质上不是"真实哈密顿量的梯度"，而是 gα = −(\text{target-overlap 标量}) × (\text{cross-overlap 导数})。 所以整个问题最终被降成：固定目标 ECG 和当前 trial ECG 之间的 overlap 及其参数导数怎么高效、稳定地算出来。
