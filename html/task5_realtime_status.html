3. 病理学 — 两种不相容的故障模式

基本困境：C 矩阵有若干种"零本征值"（物理 + gauge）和连续谱"小 sv"。任何伪逆策略都要在 (A) 过度截断造成信息丢失 和 (B) 截断不足造成 gauge 泄露 之间选。没有两全。

3.1 失效模式 A：width 重用（结构奇异）

Greedy 选 width-major 时，K=5 会选到 4 个相同 width 但不同 n 的候选。∂A, ∂B 导数几乎平行 → C 在 (A,B) 子空间精确奇异。rcond 硬截断把 5 个 sv 钉在 λTikh=1e-8 地板 → 物理方向被挤出 → 一旦 g 在被挤的方向有非零投影，dz 在另一方向暴涨。

观察：K=5 width-major 在 step 77 \|dz\|=74 崩；basis dump 显示 basis[2]=basis[3] 的 B 完全相同（Route 1 polish 后的 reflection 对称局部最小值）。

3.2 失效模式 B：±n 镜像对（边际 gauge）

将 pool 改成 |n|-major 把 ±n 对放在同 width，避开 width 重用，但引入了 x→-x 反射对称性。C 在 (basis_-n, basis_+n) 子空间有近零但非零的 sv（比如 1e-7 级）。rcond=1e-4 截不到它，Wiener 也会给 0.3× 的非零 sinv → gauge 方向获得非零 dz。

观察：K=3 interleave 从 E(0)=0.658 → E(t=0.16)=0.549，一次大 \|dz\| 事件后锁死在 K=3 manifold 的自然极值 0.549。K=5 interleave 在 step 50 崩，step 3 norm 变负 6000+。

3.3 失效模式 C：rcond 阈值穿越（阶梯跳）

无论 ordering 怎么选，C 的 sv 谱随参数演化在漂。某个 sv 从 rcond·svmax 上方漂到下方（或反向），导致该方向在伪逆里开关式切换（1/sv 突变为 0 或反过来）。dz 不连续 → E 阶梯跳。

观察：K=5 dt=1e-3 + interleave 在 1558 步中 E 从 0.679 → 0.700 → 0.728 → 1.134 → 1.582 → 1.601，每次跳对应 sv_min3[1] 从 1.01e-8 附近跨越某阈值。

3.4 失效模式 D：Tikhonov 与 Wiener 冲突

Wiener 平滑公式 sinv = sv / (sv² + λ²)：在 sv=0 严格给零（消除真 gauge），在 sv=λ 达最大放大 1/(2λ)。

但 Tikhonov λC=1e-8 把所有 sv 地板抬到 1e-8，远低于 Wiener 的 soft-shoulder λ=rcond·svmax≈1.78e-4。Wiener 看到 sv=1e-8（原本应是 0），计算 sinv = 1e-8 / (1e-16 + 3.17e-8) = 0.316 → gauge 方向被赋予 0.316 倍放大，泄露到参数空间。

观察：K=5 Wiener+Tikh 在 step 3 norm = -6164.95（不是数值小，是 u 全局符号翻转），step 5 基底 Re(A+B)=-5.19。Wiener + Tikh 一起用时 比任何单一方案都崩得快。

4. Wiener + 小 dt 为什么工作（新）

Wiener 公式 sinv = sv / (sv² + λ²) 的三个关键性质：

  • sv = 0 严格给 0：真 gauge 方向的 nullspace 精确清零（必须 λ_C=0，否则 Tikhonov 把 sv 抬到 1e-8 地板，Wiener 看成"非零", 给 0.3× 放大）
  • 连续函数：sv 在参数演化中漂过任何值时, sinv 平滑变化, 无阶梯跳
  • 有限放大：最大放大率 = 1/(2λ), 在 sv=λ 处取到（比硬 rcond 的 1/(rcond·sv_max) 小 2×）

但 Wiener 不是 TDVP ODE 的严格解——它是有偏差的近似。每步偏差量级 O(λ²/sv) 对"刚过软 shoulder"的方向, 积累 N 步得误差 O(N·dt·偏差)。

核心观察：偏差积累和 dt 成正比，所以 dt 必须足够小让偏差在 T=2π 内保持可控。
  • dt = 1e-2：开头几步的 Wiener bias 直接把状态推到错位 manifold，E 锁在 5.96 失败
  • dt = 1e-3：每步 bias × 10 小，状态保持在正确流形附近, E 几乎恒定 4000 多步 成功

5. augment 算法的 ordering 影响

Candidate pool 构造顺序决定小 K 时 greedy 选哪些基。两种尝试过的 ordering：

Width-major (旧, 现 revert) (0.3,-2), (0.3,-1), (0.3,+1), (0.3,+2), (0.5,-2)... pre_kick + (0.3,-2) + (0.3,-1) pre_kick + 4 copies all w=0.3 → 模式 A

6. 诊断工具（已加，在 realtime_tdvp.cpp 主循环每步打印）

  • cond = sv_max / sv_min，C_bar 条件数
  • sv_min3=(最小, 第二小, 第三小) — C_bar 底部 3 个奇异值，看 gauge 地板是否被 Tikhonov 拉起、有没有"边际 sv"卡在 rcond 阈值附近
  • |dz| — 合并 dz 的 L2 范数，看伪逆有没有放大失控
  • min(Re A+B)@i — 所有基底里 Gaussian 归一化因子 Re(A+B) 的最小值。接近 0 或变负 → 基底跑出物理区
  • min(Re A)@i — Re(A) 本身可以为负（复 A 容许），只要 A+B 实部正即可

7. 和 K=1 自检 (task F) 的对比

K=1 纯基态跑 628 步全部 机器零误差，证明核心 TDVP 矩阵元、∂αψ 导数、P⊥ 投影、RK4 积分全部没有 bug。Task E 的所有失败都是 K≥3 时过参数化 + 伪逆病态的后果，不是代码错误。

8. 文献对照 — 我们不是第一个撞这堵墙

Richings, Polyak, Spinlove, Worth et al., Int. Rev. Phys. Chem. 34:269 (2015) vMCG review 原话：

"When the basis set is complete, the parameter derivatives cannot be
 projected outside the space spanned by the Gaussian functions and C
 becomes zero. This pathological property means that the vMCG EOMs
 become very unstable."

他们的工程修法是 Tikhonov with ε=1e-6（业界默认），仍然没解决根本问题——只是把崩的时刻延后。

Schrader, Pedersen, Kvaal, JCP 162, 024109 (2025) 明确诊断 vMCG 的"vanishing gradient / barren plateau" 问题，并提出 Rothe 方法作为替代：不反 C 矩阵，而是每步做 Crank-Nicolson + VarPro + BFGS 最小化。他们证明 2D Hénon-Heiles 30 个 Gaussian 能复现 512×512 格点的结果。

本地 PDF：reference/Multidimensional quantum dynamics with explicitly correlated Gaussian wave packets using Rothe's method.pdf。方案：html/rothe_implementation_plan.html。

  1. ★ K=5 + Wiener + λ=0 + dt=1e-3（15 分钟，最关键）：
    ./build/ecg1d_simple --task5 --n-max 4 --n-mom 2 --T-free 6.283185307 \
    --dt 1e-3 --integrator rk4 --max-cond 1e6 \
    --no-u-split --no-evolve-B \

    K=3 已确认工作。如果 K=5 也撑完且 \|F(T)\|²>0.9, 说明 Wiener+dt=1e-3 是可扩展的工作点, 不止是对 K=3 的巧合。
  2. K=7 + 同配置（30 分钟）：
    上面 K=5 通过后, 把 K-cap 改 7。基底更丰富, F_init 更接近 1, Wiener bias 也可能更大。看能不能平衡到 \|F(T)\|²>0.9。
  3. 扫 dt（1 小时）：
    固定 K=3（或 K=5 如果通过）, 试 dt ∈ {2e-3, 5e-3, 1e-3, 5e-4}。看 Wiener 误差是否随 dt² 下降（RK4 期望）。有可能 dt=5e-4 能把 \|F(T)\|² 推到 0.95+。
  4. 多 kick 循环（MBDL 主线）（1-2 天）：
    用上面确认的工作点跑。每 kick 之间再次 Route 1 polish basis（或直接重用上步 basis）, 记录 E(t), n(k), G⁽¹⁾(z)。看多 kick 能否出 MBDL 的定性信号。即使单 kick 有 10% 损失, 多 kick 累积可能仍能看到趋势。
  5. 长期备选：Rothe 方法（非紧急）：
    见 html/rothe_implementation_plan.html。当前 Wiener+小 dt 走得通就先不切。Rothe 作为"如果 K=5,7 也上不去" 的保底方案。

    如果 K=5, K=7 也撑过 6283 步且 F(T)²>0.9, 说明 Wiener 工作点可扩展到更丰富基底。这是下一个关键里程碑。
