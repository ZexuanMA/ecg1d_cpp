2. Utility helpers (lines 25–97)

2.1 load_basis_from_csv (lines 28–68)

Reads a basis from disk. Each basis function serializes as u, then N×N complex entries of A, then N×N of B, then N entries of R, then a single integer name.

static std::vector<BasisParams> load_basis_from_csv(
    const std::string& filename, int N, int basis_n) {
    std::ifstream fin(filename);
    if (!fin.is_open()) { /* error */ return {}; }

    auto read_complex = [&]() -> Cd {            // lambda, captures fin by reference
        std::string line; std::getline(fin, line);
        std::stringstream ss(line);
        double re, im; char comma;
        ss >> re >> comma >> im;
        return Cd(re, im);
    };
    // ... uses read_complex() for u, A, B, R
}
C++ Anatomy of auto read_complex = [&]() -> Cd { ... };
  • [&] is the capture list. & means "capture all enclosing variables by reference" — so the lambda sees the same fin the outer function has. If you used [=] that would copy fin, which is forbidden for ifstream.
  • () -> Cd is the parameter list and explicit return type. The return type can be omitted (the compiler deduces it), but stating it here documents intent.
  • The lambda is stored in a variable via auto. Its actual type is an unnamed compiler-generated struct with operator() — you can't spell it, but auto can.
  • static on the free function means internal linkage: the symbol is only visible in this .cpp file. It prevents link-time name clashes and lets the compiler inline more aggressively.
Subtle The code parses with ss >> re >> comma >> im. That only works because each line has the form "3.14,1.59". If the CSV ever gains a space after the comma, this still works (stream >> skips whitespace). If it gains multiple commas, it will silently break.

2.3 NullBuffer + ScopedSilence (lines 74–97)

class NullBuffer : public std::streambuf {
protected:
    int overflow(int c) override { return c; }   // swallow all output
};

class ScopedSilence {
public:
    explicit ScopedSilence(bool enabled) : enabled_(enabled) {
        if (enabled_) { old_ = std::cout.rdbuf(&null_); }
    }
    ~ScopedSilence() { if (enabled_) { std::cout.rdbuf(old_); } }
private:
    bool enabled_ = false;
    NullBuffer null_;
    std::streambuf* old_ = nullptr;
};
RAII — Resource Acquisition Is Initialization
This is the single most important C++ idiom. A constructor grabs a resource; the matching destructor releases it. You can never forget to release, because the compiler inserts the destructor call at every scope exit (normal return, exception, break).

Here, the "resource" is the redirection of std::cout. Inside a { ScopedSilence silence(true); ... } block, std::cout prints to nowhere. When the block ends, the destructor restores the original buffer automatically. Used in run_kicked_gamma0_test to silence verbose SVM output.
C++ Why explicit? Without explicit, a line like ScopedSilence s = true; would compile and silently create one. With explicit, the caller must write ScopedSilence s(true);. Always mark single-argument constructors explicit unless you really want implicit conversion.

3. State-summary utilities (lines 99–200)

3.1 collect_u and build_cross_overlap_matrix

static VectorXcd collect_u(const std::vector<BasisParams>& basis) {
    VectorXcd u(static_cast<int>(basis.size()));
    for (int i = 0; i < u.size(); i++) u(i) = basis[i].u;
    return u;
}

Builds an Eigen column vector of linear coefficients from a basis. static_cast<int> is used because .size() returns std::size_t (unsigned), and mixing signed/unsigned triggers warnings — be explicit about the narrowing.

static MatrixXcd build_cross_overlap_matrix(
    const std::vector<BasisParams>& left,
    const std::vector<BasisParams>& right,
    const PermutationSet& perms) {
    HamiltonianTerms terms_none;                   // default init → all true!
    terms_none.kinetic = false;                    // flip them all off
    terms_none.harmonic = false;
    terms_none.delta = false;
    terms_none.gaussian = false;
    terms_none.kicking = false;
    MatrixXcd C = MatrixXcd::Zero(left.size(), right.size());
    for (int i = 0; ...)
        for (int j = 0; ...) {
            auto [h_ij, s_ij] = compute_HS_ij(left[i], right[j], perms, terms_none);
            (void)h_ij;                           // explicitly ignore
            C(i, j) = s_ij;                          // keep only overlap
        }
    return C;
}
Trick Using HamiltonianTerms to ask for "overlap only". Because compute_HS_ij always returns both the Hamiltonian matrix element and the overlap, turning every Hamiltonian term off leaves s_ij as the overlap and h_ij as zero. This is an elegant way to reuse one code path for two tasks.
C++17 Structured bindings: auto [h_ij, s_ij] = ...; unpacks a std::pair (or any struct with public fields) into two named variables. Equivalent to auto tmp = ...; auto h_ij = tmp.first; auto s_ij = tmp.second;.
Idiom (void)h_ij; is a C-era trick to tell the compiler "I know I'm not using this variable, don't warn". In modern C++ you could also write auto [_, s_ij] = ... with a stub name.

3.2 normalized_state_fidelity (lines 128–137)

static double normalized_state_fidelity(
    const VectorXcd& u_L, const MatrixXcd& S_L,
    const VectorXcd& u_R, const MatrixXcd& S_R,
    const MatrixXcd& cross) {
    double nL = (u_L.adjoint() * S_L * u_L)(0).real();
    double nR = (u_R.adjoint() * S_R * u_R)(0).real();
    if (nL <= 1e-15 || nR <= 1e-15) return 0.0;
    Cd ov = (u_L.adjoint() * cross * u_R)(0);
    return std::norm(ov) / (nL * nR);   // |<ψL|ψR>|² / (<ψL|ψL> <ψR|ψR>)
}
Eigen u.adjoint() is the conjugate transpose (Hermitian conjugate), the right thing for complex inner products. u.transpose() would only transpose, and would give you uTAu rather than u†Au.

The expression u.adjoint() * S * u is a 1×1 Eigen matrix. You extract its single element with (0), then take .real() because the inner product of a Hermitian form is real up to roundoff.
C++ std::norm(z) returns |z|², not the norm (absolute value). For |z| use std::abs(z). A common source of bugs for C++ beginners.

4.2 overlap_condition_number (lines 191–200)

static double overlap_condition_number(const MatrixXcd& S) {
    Eigen::SelfAdjointEigenSolver<MatrixXcd> es(S);
    if (es.info() != Eigen::Success) return std::numeric_limits<double>::infinity();
    double wmin = es.eigenvalues()(0);
    double wmax = es.eigenvalues()(S.rows() - 1);
    if (wmin <= 0.0) return std::numeric_limits<double>::infinity();
    return wmax / wmin;
}
Eigen SelfAdjointEigenSolver is specialized for Hermitian matrices — faster and more accurate than the general solver. Eigenvalues are returned in ascending order, so (0) is smallest, (n-1) is largest.

The ratio λmax/λmin is the condition number of S. A large condition number means some basis functions are nearly linearly dependent and numerical inversion of S will amplify round-off.

4.3 compute_first_kick_diagnostic / first_kick_objective (lines 202–230)

Apply one analytic kick to the basis, summarize observables, compare to the exact target, and reduce to a scalar "objective" to minimize via basis search. The objective is a weighted sum:

J = rel(E) + 3·|Δ⟨x²⟩| + rel(⟨p²⟩) + 0.25·|1 − fidelity| + 5·gs_penalty + 0.05·cond_ratio

This is literally the acceptance criterion in the packet search — the weights were hand-tuned to favor dynamical fidelity while still punishing ill-conditioned bases.

4.4 Momentum-label generator (lines 232–280)

Returns a curated list of integer labels (m_1, m_2, ..., m_N). Each label will later become a basis function with momentum 2·m_a·k_L on particle a. The list is hand-picked (preferred) rather than generated exhaustively — reflects physical intuition about which momentum sectors a first kick excites.

4.5 make_first_kick_image_candidate (lines 282–314) physics-critical

static RepairCandidate make_first_kick_image_candidate(
    const BasisParams& base, const std::vector<int>& labels,
    double k_L, double b_val) {
    int N = base.N();
    MatrixXcd A_new = base.A;
    MatrixXcd B_new = base.B;
    VectorXcd R_new = base.R;
    for (int a = 0; a < N; a++) {
        if (labels[a] == 0) continue;
        Cd delta_b(b_val, 0.0);
        A_new(a,a) -= delta_b;                         // shift width from A into B
        B_new(a,a) += delta_b;
        Cd q_a(0.0, 2.0 * labels[a] * k_L);         // purely imaginary → momentum phase
        Cd old_linear = 2.0 * base.B(a,a) * base.R(a);
        R_new(a) = (old_linear + q_a) / (2.0 * B_new(a,a));
    }
    ...
}

This implements the identity (see principle doc §2): a Gaussian times exp(i·q·x) can be reabsorbed into the Gaussian by shifting R to a complex value such that 2BR acquires the imaginary part iq. The kick operator multiplies the wavefunction by exp(i·2m·k_L·x), and this function constructs a basis function that represents exactly that momentum image.

4.6 generate_repair_candidates, generate_repair_packets (lines 316–382)

Both pick a few "parent" basis functions (those with the largest |u|) and cross them with a curated list of momentum labels. _candidates returns individual entries; _packets returns bundles of 4/8/12/16 candidates because rank-deficient single-entry additions often fail the conditioning gate, but a full packet covers the right momentum subspace.

C++ std::sort with a lambda comparator: 
std::sort(order.begin(), order.end(),
          [&](int lhs, int rhs) {
              return std::abs(source[lhs].u) > std::abs(source[rhs].u);

The lambda defines a strict weak ordering. Returning a > b sorts descending. Capturing by reference ([&]) gives the lambda access to source.

6.1 Building the alpha_z_list manually

std::vector<AlphaIndex> alpha_z_list;
for (int i = 0; i < basis_n; i++) alpha_z_list.push_back({1, i, 0, 0});   // u

An AlphaIndex is a 4-tuple (a1, a2, a3, a4):

  • a1 = 1: index into u (the linear coefficient)
  • a1 = 2: index into B[basis, particle]
  • a1 = 3: index into R[basis, particle]
  • a1 = 4: index into A[basis, i, j] (the upper-triangular entries)

This list tells the TDVP solver which variational parameters are dynamical coordinates. The later function build_alpha_z_list (§7) automates building this list based on the SolverConfig.

7. build_alpha_z_list (lines 638–664)

static std::vector<AlphaIndex> build_alpha_z_list(
    int basis_n, int N, const SolverConfig& cfg) {
    std::vector<AlphaIndex> out;
    for (int i = 0; i < basis_n; i++) out.push_back({1, i, 0, 0});   // u always
    if (cfg.optimize_B) for(...) out.push_back({2, i, j, 0});
    if (cfg.optimize_R) for(...) out.push_back({3, i, j, 0});
    if (cfg.optimize_A)
        for (int i = 0; i < basis_n; i++)
            for (int j = 0; j < N; j++)
                for (int k = j; k < N; k++) // upper-triangular only
                    out.push_back({4, i, j, k});
    return out;
}
Why upper-triangular A? A is symmetric, so A(i,j) = A(j,i). Treating both as independent would double-count gradient components and make C rank-deficient.

10. run_kicked_gamma0_test (lines 918–1278)

This is the most intricate function in the file. It assembles a non-TDVP real-time pipeline that gives better long-time stability than raw TDVP for the γ=0 (non-interacting) kicked rotor benchmark. Outline:

  1. Phase 1 (silenced): SVM → stochastic refine → imaginary-time TDVP for the ground state. A ScopedSilence suppresses the verbose output.
  2. Phase 2 (basis augmentation): call augment_basis_with_momentum to add momentum-carrying basis functions indexed by m ∈ {-n_mom, ..., n_mom}.
  3. Phase 2.5 (packet repair): try bundles of additional basis functions — each bundle approximates a truncated kick image of a source basis function. Accept at most two bundles that improve the first-kick objective J.
  4. Ground-state consistency checks: cross-basis fidelity (does the augmented basis still represent the ground state?), variance sanity, and a "no-kick free test" (propagate free in the fixed basis for 5 periods and verify no drift).
  5. Phase 3 (kicked dynamics): loop n_kicks times:
      (a) apply_analytic_kick — instantaneous basis-projection kick (no TDVP, no finite T_pulse).
      (b) free_evolve_fixed_basis — exact generalized-eigenbasis propagation over one period.
  6. Collect observables, print a per-kick comparison against kicked_exact_1particle, emit a PASS/FAIL summary.
Why this path exists
TDVP drifts over long evolution because every step carries roundoff. In contrast, the augmented-basis approach keeps the basis fixed after initial construction and propagates only the coefficient vector u. Inside the fixed finite-dimensional subspace, free propagation and kick are each implemented as exact operators, so the only error is the basis-truncation error — it does not accumulate over time.

10.1 Instructive C++ patterns in this function

(a) Lambda with immediate invocation for scoped side effects (line 943)

auto svm = [&]() {
    ScopedSilence silence(quiet_setup);

This is an IIFE (Immediately Invoked Function Expression). The lambda runs exactly once, creates a ScopedSilence locally (so silencing is scoped to the SVM call only), and returns the result. As soon as the lambda finishes, silence destructs and std::cout is restored. The auto svm then captures the return value.

(b) std::move for ownership transfer (line 1089–1091)

best_basis = std::move(trial_basis);
best_H     = std::move(H_trial);
best_S     = std::move(S_trial);
C++11 std::move casts its argument to an rvalue, enabling move semantics: the destination steals the internal storage (pointers) instead of copying. After std::move, the moved-from object is in a valid-but-unspecified state — the rule is "you may destroy or assign to it; don't read its value". For large MatrixXcd copies this saves gigabytes of memory traffic.

(c) Tolerance gating via lambdas is not used here, but manual filters are

for (int k = old_size; k < trial_basis.size(); k++) {

Two-stage filter: skip any packet with a near-duplicate function inside (fidelity > 0.995), then skip any packet pushing condition number over max_cond.

Key iomanip rules:

  • setw(n) affects only the next item (sticky off).
  • setprecision(n) is sticky (applies until changed).
  • std::scientific / std::fixed / std::defaultfloat are sticky format modes — always pair scientific with a later defaultfloat to restore.

11. run_kick_convergence_test — Trotter split (lines 1282–1476)

Studies how TDVP-approximated kicks converge to the exact instantaneous-kick answer as T_pulse → 0. Steps:

  1. Step A: build a small hand-picked N=1 ground-state basis (three Gaussians with widths 0.2, 0.5, 2.0).
  2. Sanity: evolve freely for 5 periods and check energy/norm drift.
  3. Step B: compute exact reference kick energies via kicked_exact_1particle.
  4. Step C: direct baseline — apply analytic kick + exact propagation, no TDVP.
  5. Step D: sweep T_pulse ∈ {0.5, 0.2, 0.1, 0.05, 0.02, 0.01}. For each, TDVP-evolve: (free window, T_period − T_pulse) then (pulse window, T_pulse) × n_kicks.
  6. Step E: print a convergence table with apparent order-of-accuracy, comparing with Step C.

11.2 dt_kick selection

double scale = 1.0 / T_pulse;
double dt_kick = std::min(T_pulse / 50.0, 0.05 / (scale * kappa));
if (dt_kick < 1e-6) dt_kick = 1e-6;
int n_kick_steps = static_cast<int>(std::ceil(T_pulse / dt_kick));

The constraint dt·scale·kappa ≈ 0.05 keeps the RK4 integrator stable during the strong kick pulse. Without it, for small T_pulse, the effective strength kappa·scale = kappa/T_pulse blows up and RK4 diverges.

For errors scaling like err ∝ T_pulsep, p = log(e₂/e₁) / log(T₂/T₁). Expect p≈1 for first-order Trotter.

15.4 The aliasing trap pitfall

Wrong (found in our own codebase history, see EXPERIMENT_S_CONDITIONING.zh.md): 
A = 0.5 * (A + A.transpose());                    // does NOT symmetrize!
Right: 
A = (0.5 * (A + A.transpose())).eval();
Eigen evaluates expressions lazily. Without .eval(), A is being read from and written to simultaneously, producing wrong entries. The fix forces evaluation into a temporary before assignment.
